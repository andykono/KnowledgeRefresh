# Singleton Pattern

[[1. Patterns.md]]

Ensures a class has only one instance and provides global access to it—perfect for shared resources like API clients or auth managers.

## Classic Class Implementation

```typescript
class ApiClient {
  private static instance: ApiClient | null = null;

  private constructor(private baseUrl: string) {}

  static getInstance(): ApiClient {
    if (!ApiClient.instance) {
      ApiClient.instance = new ApiClient('https://api.example.com');
    }
    return ApiClient.instance;
  }

  async get<T>(path: string): Promise<T> {
    const res = await fetch(`${this.baseUrl}${path}`);
    return res.json() as Promise<T>;
  }

  async post<T>(path: string, data: unknown): Promise<T> {
    const res = await fetch(`${this.baseUrl}${path}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    });
    return res.json() as Promise<T>;
  }
}

// Usage everywhere
const api = ApiClient.getInstance();
```

## React Integration Example

```typescript
// services/apiClient.ts
class ApiClient {
  private static instance: ApiClient | null = null;

  private constructor(private baseUrl: string) {}

  static getInstance(): ApiClient {
    if (!ApiClient.instance) {
      ApiClient.instance = new ApiClient(process.env.REACT_APP_API_URL || '');
    }
    return ApiClient.instance;
  }

  async get<T>(path: string): Promise<T> {
    const res = await fetch(`${this.baseUrl}${path}`);
    return res.json();
  }
}

export const apiClient = ApiClient.getInstance();

// hooks/useUsers.ts
import { useState, useEffect } from 'react';
import { apiClient } from '../services/apiClient';

interface User {
  id: number;
  name: string;
  email: string;
}

export function useUsers() {
  const [users, setUsers] = useState<User[]>([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setLoading(true);
    apiClient
      .get<User[]>('/users')
      .then(setUsers)
      .catch(setError)
      .finally(() => setLoading(false));
  }, []);

  return { users, loading, error };
}

// components/UsersList.tsx
function UsersList() {
  const { users, loading, error } = useUsers();

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name} ({user.email})</li>
      ))}
    </ul>
  );
}
```

## Modern Alternative: Module-Level Singleton

```typescript
// services/apiClient.ts - Modern approach (no class needed!)
export class ApiClient {
  constructor(private baseUrl: string) {}

  async get<T>(path: string): Promise<T> {
    const res = await fetch(`${this.baseUrl}${path}`);
    return res.json();
  }
}

// Singleton via module export
export const apiClient = new ApiClient(
  process.env.REACT_APP_API_URL || 'https://api.example.com'
);
```

## When to Use Singleton in React

✅ **Good use cases:**
- API clients (shared across app)
- Logger service
- Auth service
- Theme manager
- Analytics tracker

❌ **Avoid:**
- Global state (use Context API or Redux instead)
- Mutable shared objects (causes bugs)
- Testing (hard to mock, creates tight coupling)

## Testing Singleton

```typescript
// __tests__/apiClient.test.ts
describe('ApiClient', () => {
  it('returns same instance', () => {
    const instance1 = ApiClient.getInstance();
    const instance2 = ApiClient.getInstance();
    expect(instance1).toBe(instance2);
  });

  it('should fetch users', async () => {
    const api = ApiClient.getInstance();
    // Mock fetch
    global.fetch = jest.fn(() => Promise.resolve({
      json: () => Promise.resolve([{ id: 1, name: 'John' }])
    }));

    const users = await api.get('/users');
    expect(users).toHaveLength(1);
  });
});
```

## Common Pitfalls

⚠️ **Thread safety:** JavaScript is single-threaded, but lazy initialization works fine
⚠️ **Testing issues:** Hard to reset singleton between tests → use factory function instead
⚠️ **Dependency Injection:** Prefer passing instance as prop than global access
