# Architectural Patterns

[[../README.md]]

Архитектурные паттерны определяют **общую структуру приложения** на высоком уровне. В отличие от дизайн-паттернов (которые решают локальные проблемы), архитектурные паттерны влияют на организацию кода всего проекта.

---

## What are Architectural Patterns?

Архитектурные паттерны:
- ✅ Определяют **слои** и **структуру папок**
- ✅ Решают проблемы **масштабирования**
- ✅ Улучшают **тестируемость** и **поддерживаемость**
- ✅ Облегчают **командную работу**

---

## Key Differences: Patterns vs Architecture

| Аспект | Design Patterns | Architectural Patterns |
| :-- | :-- | :-- |
| **Масштаб** | Отдельные классы/функции | Целое приложение |
| **Время применения** | При написании кода | При планировании проекта |
| **Фокус** | Как создавать/использовать объекты | Как организовать код |
| **Примеры** | Singleton, Factory, Observer | MVC, MVVM, Clean Architecture |

## Quick Comparison

| Pattern                   | Best For                        | Team Size | Complexity | React Fit |
| :------------------------ | :------------------------------- | :-------- | :--------- | :-------- |
| **Component-Based**       | Simple apps, prototypes         | 1-2       | Low        | ⭐⭐⭐       |
| **MVVM**                  | Medium apps with state          | 3-5       | Medium     | ⭐⭐⭐       |
| **Clean Architecture**    | Enterprise, testability         | 5+        | High       | ⭐⭐        |
| **Onion Architecture**    | Domain-driven, complex business | 5+        | High       | ⭐⭐⭐       |
| **Feature-Sliced Design** | Scalable modular apps           | 5+        | Medium     | ⭐⭐⭐⭐      |

## When to Use Each

### Start with Component-Based
- Learning React
- Small projects (< 5 components)
- Prototyping

### Move to MVVM + Hooks
- Growing teams (3-5 devs)
- Complex state management
- Multiple pages/features

### Adopt FSD (Recommended)
- Scaling teams (5+ devs)
- Multiple features
- Long-term maintenance

### Add Clean Architecture / Onion Architecture
- Enterprise requirements
- High testability needed
- Multiple backends/integrations
- Domain-driven design (Onion)

---

## Recommended Learning Path for React

```
Week 1: Component-Based Architecture
   ↓
Week 2-3: MVVM with Hooks + Context
   ↓
Week 4-5: Feature-Sliced Design (Recommended)
   ↓
Week 6+: Clean Architecture OR Onion Architecture + SOLID
```

**Note:** For most React projects, **Feature-Sliced Design (FSD)** is recommended. Clean Architecture and Onion Architecture are useful for **enterprise-grade, complex domain-driven applications**.
