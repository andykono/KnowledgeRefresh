# Closures

[[5. FED/3. JavaScript/1. Core concepts/2. closures]] · [[4. Common/4. CallApplyBind/1. CallApplyBind]] · [[4. Common/2. Debounce/1. Debounce]]

## Определение

**Closure** (замыкание) — функция, которая сохраняет доступ к переменным из внешней области видимости (лексического окружения) даже после того, как эта внешняя функция завершила работу.

Это не отдельная сущность, а **естественное поведение JavaScript**, благодаря лексической области видимости (lexical scoping).

### Формула

```
Closure = Function + Lexical Environment
```

## 1. Базовый пример: Приватные данные

Замыкание позволяет создавать приватные переменные, которые недоступны снаружи.

### Пример 1: Счетчик

```javascript
function createCounter() {
  let count = 0;  // «приватная» переменная, недоступна снаружи

  return function increment() {
    return ++count;  // increment "помнит" count из createCounter
  };
}

const counter = createCounter();
console.log(counter());  // 1
console.log(counter());  // 2
console.log(counter());  // 3

console.log(count);  // ReferenceError: count is not defined (приватная!)
```

### Как это работает:

1. Вызываем `createCounter()`, она создает `count = 0`
2. Возвращается функция `increment`, которая имеет доступ к `count`
3. `createCounter()` завершается, но `increment` остаётся в памяти
4. При каждом вызове `increment()` получает доступ к исходному `count` (он не уничтожается!)

### Пример 2: Аккумулятор

```javascript
function createAccumulator(initialValue = 0) {
  let total = initialValue;

  return {
    add: (value) => total += value,
    subtract: (value) => total -= value,
    get: () => total,
    reset: () => (total = initialValue)
  };
}

const acc = createAccumulator(100);
acc.add(50);        // total = 150
acc.subtract(20);   // total = 130
console.log(acc.get());  // 130
acc.reset();        // total = 100
console.log(acc.get());  // 100
```

## 2. Stale Closure (Устаревшее замыкание)

Замыкание может "захватить" переменную в момент её создания. Если эта переменная позже изменится, замыкание будет видеть новое значение.

### Пример 1: Проблема с var в цикле

```javascript
// НЕПРАВИЛЬНО:
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i);  // 3 3 3 (видит финальное значение i)
  }, 0);
}

// Почему? var имеет function scope, не block scope
// К моменту вызова setTimeout переменная i уже равна 3
```

**Решение 1: Использовать let (рекомендуется)**

```javascript
// ПРАВИЛЬНО:
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i);  // 0 1 2 (каждая итерация — своя i)
  }, 0);
}

// let имеет block scope, на каждой итерации создаётся новая переменная
```

**Решение 2: IIFE (если нужен var)**

```javascript
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => {
      console.log(j);  // 0 1 2
    }, 0);
  })(i);  // передаём i как параметр, создаём новую переменную j
}
```

### Пример 2: Замыкание в array.map

```javascript
const functions = [];

for (let i = 0; i < 3; i++) {
  functions.push(() => i);
}

console.log(functions[0]());  // 0
console.log(functions[1]());  // 1
console.log(functions[2]());  // 2

// С var было бы 3 3 3
```

## 3. Factory Pattern (Фабрика)

Замыкание идеально подходит для создания фабрик, которые производят объекты с приватным состоянием.

### Пример 1: Фабрика пользователей

```javascript
function createUser(name, email) {
  let balance = 0;  // приватное поле

  return {
    getName: () => name,
    getEmail: () => email,
    addMoney: (amount) => {
      if (amount > 0) {
        balance += amount;
        return `Added ${amount}, balance: ${balance}`;
      }
      return 'Invalid amount';
    },
    getBalance: () => balance
  };
}

const user1 = createUser('Alice', 'alice@example.com');
const user2 = createUser('Bob', 'bob@example.com');

console.log(user1.getName());        // Alice
console.log(user1.addMoney(100));    // Added 100, balance: 100
console.log(user1.getBalance());     // 100

console.log(user2.getBalance());     // 0 (независимое состояние)

// Невозможно получить доступ к balance напрямую:
console.log(user1.balance);  // undefined
```

### Пример 2: Фабрика функций с конфигурацией

```javascript
function createMultiplier(factor) {
  return function(number) {
    return number * factor;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);
const tenTimes = createMultiplier(10);

console.log(double(5));      // 10
console.log(triple(5));      // 15
console.log(tenTimes(5));    // 50

// Каждая функция "помнит" свой factor
```

## 4. Module Pattern

Замыкание используется для создания приватного пространства имён (модулей).

### Пример 1: Простой модуль

```javascript
const Calculator = (function() {
  // Приватные переменные
  let history = [];

  // Приватная функция
  function log(operation, result) {
    history.push({ operation, result, timestamp: new Date() });
  }

  // Публичный API (возвращаемый объект)
  return {
    add(a, b) {
      const result = a + b;
      log('add', result);
      return result;
    },

    subtract(a, b) {
      const result = a - b;
      log('subtract', result);
      return result;
    },

    getHistory() {
      return history;
    },

    clearHistory() {
      history = [];
    }
  };
})();

console.log(Calculator.add(5, 3));        // 8
console.log(Calculator.subtract(10, 2));  // 8
console.log(Calculator.getHistory());     // [{ operation: 'add', result: 8, ... }, ...]

// Нельзя обратиться к history напрямую:
console.log(Calculator.history);  // undefined
```

### Пример 2: Модуль с конфигурацией

```javascript
const UserManager = (function(config) {
  const users = [];
  const maxUsers = config.maxUsers || 100;

  return {
    add(name) {
      if (users.length >= maxUsers) {
        return `Cannot add user, max ${maxUsers} reached`;
      }
      users.push({ name, id: users.length + 1 });
      return `User ${name} added`;
    },

    getAll() {
      return users;
    },

    count() {
      return users.length;
    }
  };
})({ maxUsers: 5 });

console.log(UserManager.add('Alice'));    // User Alice added
console.log(UserManager.add('Bob'));      // User Bob added
console.log(UserManager.count());         // 2
```

## 5. Currying (Каррирование)

Преобразование функции многими аргументами в цепочку функций с одним аргументом.

### Пример 1: Простое каррирование

```javascript
// Обычная функция
function add(a, b, c) {
  return a + b + c;
}

// Каррированная версия
function curriedAdd(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}

console.log(curriedAdd(1)(2)(3));  // 6

// Или с arrow функциями (короче):
const curriedAdd2 = a => b => c => a + b + c;
console.log(curriedAdd2(1)(2)(3));  // 6
```

### Пример 2: Partial application с каррированием

```javascript
function multiply(a, b, c) {
  return a * b * c;
}

const curriedMultiply = a => b => c => multiply(a, b, c);

// Можно "заморозить" отдельные аргументы
const multiplyBy2 = curriedMultiply(2);
const multiplyBy2By3 = multiplyBy2(3);

console.log(multiplyBy2By3(4));  // 2 * 3 * 4 = 24
```

### Пример 3: Валидатор с каррированием

```javascript
const createValidator = (minLength) => (maxLength) => (value) => {
  if (value.length < minLength || value.length > maxLength) {
    return `Length must be between ${minLength} and ${maxLength}`;
  }
  return 'Valid';
};

const userNameValidator = createValidator(3)(20);
const passwordValidator = createValidator(8)(128);

console.log(userNameValidator('ab'));      // Length must be between 3 and 20
console.log(userNameValidator('alice'));   // Valid
console.log(passwordValidator('pass'));    // Length must be between 8 and 128
console.log(passwordValidator('strongpass123'));  // Valid
```

## 6. Декораторы с замыканием

Замыкание идеально для создания функций-оберток.

### Пример 1: Кэширование результатов

```javascript
function memoize(fn) {
  const cache = {};  // приватный кэш

  return function(...args) {
    const key = JSON.stringify(args);

    if (key in cache) {
      console.log(`Getting from cache: ${key}`);
      return cache[key];
    }

    console.log(`Computing: ${key}`);
    const result = fn.apply(this, args);
    cache[key] = result;
    return result;
  };
}

function expensiveCalculation(n) {
  let sum = 0;
  for (let i = 0; i < n * 1000000; i++) {
    sum += i;
  }
  return sum;
}

const memoized = memoize(expensiveCalculation);

console.time('First call');
memoized(10);  // Computing
console.timeEnd('First call');

console.time('Second call (cached)');
memoized(10);  // Getting from cache
console.timeEnd('Second call (cached)');
```

### Пример 2: Logger

```javascript
function logCalls(fn, name = fn.name) {
  let callCount = 0;

  return function(...args) {
    callCount++;
    console.log(`[${name}] Call #${callCount} with args:`, args);
    const result = fn.apply(this, args);
    console.log(`[${name}] Returned:`, result);
    return result;
  };
}

const loggedSum = logCalls((a, b) => a + b, 'sum');

loggedSum(5, 3);  // [sum] Call #1 with args: [5, 3]
loggedSum(10, 2);  // [sum] Call #2 with args: [10, 2]
```

### Пример 3: Rate limiter

```javascript
function rateLimit(fn, limit = 3) {
  let callCount = 0;
  let lastResetTime = Date.now();

  return function(...args) {
    const now = Date.now();

    // Сбросить счетчик через секунду
    if (now - lastResetTime > 1000) {
      callCount = 0;
      lastResetTime = now;
    }

    if (callCount < limit) {
      callCount++;
      return fn.apply(this, args);
    } else {
      throw new Error(`Rate limit exceeded: ${limit} calls per second`);
    }
  };
}

const limitedFetch = rateLimit(async (url) => {
  return fetch(url);
}, 2);

try {
  limitedFetch('url1');  // OK
  limitedFetch('url2');  // OK
  limitedFetch('url3');  // Error: Rate limit exceeded
} catch (e) {
  console.error(e.message);
}
```

## 7. Замыкание в React и Callbacks

Замыкание часто вызывает проблемы в React, если не понимать его правильно.

### Пример 1: Stale closure в React useEffect

```javascript
// ❌ НЕПРАВИЛЬНО:
function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      // Замыкание захватило первое значение count (0)
      console.log(`Count is ${count}`);  // Всегда будет "Count is 0"
    }, 1000);

    return () => clearInterval(timer);
  }, []);  // Пустой dependency array!

  return (
    <>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </>
  );
}

// ✅ ПРАВИЛЬНО:
function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      console.log(`Count is ${count}`);  // Обновляется каждый раз
    }, 1000);

    return () => clearInterval(timer);
  }, [count]);  // Добавить count в зависимости

  return (
    <>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </>
  );
}
```

### Пример 2: Обработчик клика с замыканием

```javascript
function TodoList() {
  const [todos, setTodos] = useState([]);

  const handleDelete = (id) => {
    // Замыкание "помнит" id
    setTodos(todos.filter(todo => todo.id !== id));
  };

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>
          {todo.text}
          <button onClick={() => handleDelete(todo.id)}>Delete</button>
        </li>
      ))}
    </ul>
  );
}

// handleDelete создаёт замыкание для каждого id
// Это нормально и ожидаемо в React
```

## 8. Memory Leaks с замыканиями

Замыкания могут удерживать большие объекты в памяти.

### Пример: Утечка памяти

```javascript
// ❌ Утечка памяти:
function createBigObject() {
  const largeArray = new Array(1000000).fill('data');
  let count = 0;

  return () => {
    // Замыкание удерживает largeArray в памяти!
    return ++count;
  };
}

const counter = createBigObject();  // largeArray не удаляется из памяти

// ✅ Исправление:
function createSmallCounter() {
  let count = 0;

  return {
    increment: () => ++count,
    getCount: () => count
  };
}

const counter = createSmallCounter();  // Минимум данных в памяти
```

## 9. Практические примеры

### Пример 1: Event emitter с замыканием

```javascript
function createEventEmitter() {
  const listeners = {};

  return {
    on(event, callback) {
      if (!listeners[event]) {
        listeners[event] = [];
      }
      listeners[event].push(callback);
    },

    emit(event, data) {
      if (listeners[event]) {
        listeners[event].forEach(callback => callback(data));
      }
    },

    off(event, callback) {
      if (listeners[event]) {
        listeners[event] = listeners[event].filter(cb => cb !== callback);
      }
    }
  };
}

const emitter = createEventEmitter();

emitter.on('login', (user) => {
  console.log(`${user} logged in`);
});

emitter.emit('login', 'Alice');  // Alice logged in
```

### Пример 2: API wrapper с конфигурацией

```javascript
function createAPIClient(baseURL, defaultHeaders = {}) {
  const headers = defaultHeaders;

  return {
    get(endpoint) {
      return fetch(`${baseURL}${endpoint}`, { headers });
    },

    post(endpoint, data) {
      return fetch(`${baseURL}${endpoint}`, {
        method: 'POST',
        headers: { ...headers, 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
      });
    },

    setHeader(key, value) {
      headers[key] = value;
    }
  };
}

const apiClient = createAPIClient('https://api.example.com', {
  'Authorization': 'Bearer token'
});

apiClient.get('/users');  // Использует baseURL и headers из замыкания
```

## Резюме

- ✅ **Closure** — функция + её лексическое окружение
- ✅ **Приватные данные** — замыкание скрывает переменные от внешнего доступа
- ✅ **Stale closure** — помни о значении переменных в момент создания функции
- ✅ **Module pattern** — используй IIFE для создания модулей
- ✅ **Currying** — преобразование функций с несколькими аргументами
- ✅ **Декораторы** — используй замыкание для логирования, кэширования, rate limiting
- ✅ **React** — понимай dependency arrays в useEffect чтобы избежать stale closures
- ✅ **Memory leaks** — будь осторожен с большими объектами в замыканиях
