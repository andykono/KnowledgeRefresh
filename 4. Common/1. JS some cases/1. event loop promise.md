## #1 — Promise Executor & Microtask Queue

[[4. Common/Event loop]] · [[5. FED/3. JavaScript/1. Core concepts/4. event loop]] · [[5. FED/3. JavaScript/2. ES6 features/1. Promises, async-await]]

### Код
```js
// 1
console.log('start');

const promise1 = new Promise((resolve, reject) => {
    console.log(1)
    resolve(2)
})

promise1.then(res => {
    console.log(res)
})

console.log('end');

// start 1 end 2
```

### Вывод
```
start → 1 → end → 2
```

### Почему?

| Шаг | Что происходит | Тип |
|-----|---------------|-----|
| `console.log('start')` | Выполняется сразу | Синхронно |
| `new Promise(executor)` | Executor запускается немедленно | Синхронно |
| `console.log(1)` | Внутри executor — тоже сразу | Синхронно |
| `resolve(2)` | Переводит Promise в `fulfilled`, планирует `.then` | Синхронно |
| `.then(res => ...)` | Регистрирует колбэк, уходит в **микротаск очередь** | Асинхронно |
| `console.log('end')` | Выполняется сразу | Синхронно |
| `console.log(res)` | Call stack пуст → микротаски выполняются | Асинхронно |

### Ключевые правила

- **Executor** (`new Promise(fn)`) — всегда запускается **синхронно**
- **`.then` / `.catch`** — всегда уходят в **микротаск очередь**, даже если Promise уже resolved
- Микротаски выполняются **после** текущего синхронного кода, но **до** следующего макротаска

### Три состояния Promise

```
pending  →  resolve(value)  →  fulfilled ✅
pending  →  reject(error)   →  rejected  ❌
```

> Состояние меняется **только один раз**. Повторный вызов `resolve` или `reject` игнорируется.

### Порядок очередей

```
Синхронный код
      ↓
Микротаски (.then, .catch, .finally, queueMicrotask)
      ↓
Макротаски (setTimeout, setInterval, setImmediate)
```
