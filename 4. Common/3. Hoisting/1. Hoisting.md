# Hoisting

[[5. FED/3. JavaScript/1. Core concepts/5. hoisting]] · [[4. Common/1. EventLoop/1. EventLoop]] · [[5. FED/3. JavaScript/1. Core concepts/2. closures]]

## Определение

**Hoisting** — «поднятие» (в переводе). На этапе создания выполняемого контекста JavaScript движок сканирует код и поднимает все объявления функций и переменных в начало своей области видимости ещё до выполнения кода строка за строкой.

### Важно знать

- Поднимаются **объявления**, но **не инициализация** (для `var`)
- Разное поведение для `var`, `let`, `const`
- Function declarations полностью hoisted, можно вызывать до объявления
- Function expressions не поднимаются

---

## 1. var Hoisting

Объявление **поднимается**, присваивание — **нет**. Переменная инициализируется с `undefined`.

### Пример 1: Базовый случай

```javascript
console.log(x);  // undefined (не ReferenceError!)
var x = 5;
console.log(x);  // 5
```

**Как видит JavaScript:**
```javascript
var x;           // объявление поднято
console.log(x);  // undefined
x = 5;           // присваивание остаётся на месте
console.log(x);  // 5
```

### Пример 2: Без явного присваивания

```javascript
console.log(name);  // undefined
var name;
console.log(name);  // undefined (остаётся undefined)
```

### Пример 3: Function scope

```javascript
function test() {
  console.log(x);  // undefined (поднято внутри функции)
  var x = 10;
  console.log(x);  // 10
}

test();

console.log(x);  // ReferenceError: x is not defined (вне функции нет)
```

Hoisting **не пересекает границы функций** — `var` поднимается только внутри своей функции.

### Пример 4: var в цикле (частая ошибка)

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// Вывод: 3 3 3
// Почему? Потому что var имеет function scope, а не block scope
// К моменту выполнения timeout i уже равен 3

// Как видит JavaScript:
var i;
for (i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);  // все callback'и видят один и тот же i
}
```

**Решение с let:**
```javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// Вывод: 0 1 2
// let имеет block scope, на каждой итерации создаётся новая i
```

## 2. let и const Hoisting

Поднимаются, но **не инициализируются**. До строки объявления доступ вызовет `ReferenceError`.

### Temporal Dead Zone (TDZ)

**Temporal Dead Zone** — период между началом scope и строкой объявления переменной. Если обратиться к переменной в TDZ, будет ошибка.

```javascript
console.log(a);  // ReferenceError: Cannot access 'a' before initialization
let a = 1;
```

### Пример 1: let

```javascript
console.log(x);  // ReferenceError (не undefined!)
let x = 5;
console.log(x);  // 5
```

### Пример 2: const

```javascript
console.log(y);  // ReferenceError
const y = 10;
console.log(y);  // 10
```

### Пример 3: Блочная область видимости

```javascript
{
  console.log(a);  // ReferenceError: a находится в TDZ
  let a = 5;       // объявление выходит из TDZ
  console.log(a);  // 5
}

console.log(a);  // ReferenceError: a не существует вне блока
```

### Пример 4: Nested scopes

```javascript
let x = 'outer';

{
  console.log(x);  // ReferenceError: x is in TDZ (shadowing)
  let x = 'inner';
}

// Почему ReferenceError? Потому что внутри блока объявлена новая let x,
// она поднята, но находится в TDZ до строки объявления.
// Возможная версия:
let x = 'outer';

{
  // console.log(x);  // Если раскомментировать - ReferenceError
  // Потому что видит локальное объявление let x ниже (shadow)
  let x = 'inner';
  console.log(x);  // 'inner'
}

console.log(x);  // 'outer'
```

## 3. Function Declaration Hoisting

**Function declarations полностью поднимаются** — можно вызывать до объявления.

### Пример 1: Вызов до объявления

```javascript
sayHi();  // 'Hello!' — работает!

function sayHi() {
  console.log('Hello!');
}

sayHi();  // 'Hello!' — тоже работает
```

**Как видит JavaScript:**
```javascript
// Function declaration полностью поднята
function sayHi() {
  console.log('Hello!');
}

sayHi();  // работает
sayHi();  // работает
```

### Пример 2: Function expression (не поднимается)

```javascript
greet();  // TypeError: greet is not a function

var greet = function() {
  console.log('Hi');
};
```

**Почему TypeError?** Потому что `var greet;` поднимается с `undefined`, а не сама функция.

```javascript
greet();  // TypeError: greet is not a function

var greet = function() {  // это просто присваивание
  console.log('Hi');
};

greet();  // работает
```

### Пример 3: Arrow function (функция-выражение)

```javascript
hello();  // TypeError: hello is not a function

const hello = () => {
  console.log('Hey');
};

hello();  // работает
```

Arrow function — это выражение, не declaration. Поднимается как переменная `const` (в TDZ до строки объявления).

## 4. Сравнение var, let, const

| | `var` | `let` | `const` |
|-----|-------|-------|--------|
| **Scope** | Function | Block `{}` | Block `{}` |
| **Hoisting** | Поднимается, инициализируется `undefined` | Поднимается, TDZ до объявления | Поднимается, TDZ до объявления |
| **Повторное объявление** | Да | Нет | Нет |
| **Переинициализация** | Да | Да | **Нет** |
| **Пример TDZ** | ❌ No | ✅ Yes | ✅ Yes |

### Таблица поведения:

```javascript
// var
console.log(v);  // undefined
var v = 1;
var v = 2;       // OK
v = 3;           // OK

// let
console.log(l);  // ReferenceError
let l = 1;
let l = 2;       // SyntaxError: Identifier 'l' has already been declared
l = 3;           // OK

// const
console.log(c);  // ReferenceError
const c = 1;
const c = 2;     // SyntaxError: Identifier 'c' has already been declared
c = 3;           // TypeError: Assignment to constant variable
```

## 5. Практические примеры ошибок

### Ошибка 1: Забывчивый разработчик

```javascript
function processData() {
  if (condition) {
    var result = 'success';
  }
  console.log(result);  // undefined (если condition false)
}

// Правильно с let:
function processData() {
  if (condition) {
    let result = 'success';
  }
  console.log(result);  // ReferenceError (обнаружим ошибку)
}
```

### Ошибка 2: setTimeout в цикле

```javascript
// НЕПРАВИЛЬНО
const buttons = document.querySelectorAll('button');
for (var i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener('click', () => {
    console.log(`Кнопка ${i}`);  // Все будут выводить одно и то же число
  });
}

// ПРАВИЛЬНО 1: использовать let
for (let i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener('click', () => {
    console.log(`Кнопка ${i}`);  // Каждый обработчик получит свой i
  });
}

// ПРАВИЛЬНО 2: IIFE (если нужен var)
for (var i = 0; i < buttons.length; i++) {
  (function(index) {
    buttons[index].addEventListener('click', () => {
      console.log(`Кнопка ${index}`);
    });
  })(i);
}
```

### Ошибка 3: const и мутируемые объекты

```javascript
const person = { name: 'Alice' };

person.name = 'Bob';  // OK — меняем свойство
console.log(person.name);  // 'Bob'

person = { name: 'Charlie' };  // TypeError — переприсваивание запрещено
```

`const` запрещает переприсваивание, но не запрещает мутацию свойств объекта.

## 6. Временная мертвая зона (TDZ) — детально

```javascript
function test() {
  // TDZ начинается здесь

  console.log(x);  // ReferenceError: Cannot access 'x' before initialization

  // TDZ продолжается до строки ниже
  let x = 5;
  // TDZ заканчивается здесь

  console.log(x);  // 5 (теперь можем обращаться)
}

test();
```

### TDZ с параметрами функции

```javascript
function outer(x = inner()) {
  function inner() {
    return x;  // ReferenceError: x находится в TDZ
  }
  return x;
}
```

Значение параметра по умолчанию вычисляется в своём scope, где `x` ещё не определён.

## Резюме

- ✅ Всегда используй `let` или `const`, а не `var`
- ✅ `const` по умолчанию, `let` если нужна переинициализация
- ✅ Понимай TDZ — это защита от ошибок
- ✅ Function declarations полностью hoisted, можно вызывать до объявления
- ✅ Function expressions не poднимаются как функции
