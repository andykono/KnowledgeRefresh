## Core Concepts

### Why TypeScript for React
- **Props validation** at compile time instead of runtime
- **Autocomplete** for props, state, and event handlers
- **Refactoring safety** when renaming or changing shapes
- **Documentation** via types as living specs

### Key TypeScript + React Patterns
| Pattern | Use For |
| :-- | :-- |
| Props interfaces | Component props with required/optional fields |
| Generic components | Reusable components with typed data |
| Discriminated unions | useReducer actions, state machines |
| Utility types | Partial, Pick, Omit for prop variations |
