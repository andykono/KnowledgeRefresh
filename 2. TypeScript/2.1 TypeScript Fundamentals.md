Core TypeScript concepts: basic types, interfaces vs types, generics, utility types, unions, intersections, and type narrowing.

## Basic Types

```typescript
// Primitives
const name: string = "Alice";
const age: number = 30;
const active: boolean = true;
const nothing: null = null;
const notDefined: undefined = undefined;

// Arrays
const ids: number[] = [1, 2, 3];
const names: Array<string> = ["a", "b", "c"];

// Tuples (fixed-length, typed positions)
const tuple: [string, number] = ["Alice", 30];
const [userName, userAge] = tuple;

// Objects
const user: { id: number; name: string } = { id: 1, name: "Alice" };

// Functions
const add = (a: number, b: number): number => a + b;
const greet = (name: string): void => console.log(`Hello ${name}`);
```

**Prefer `interface` for object shapes** and `type` for unions, intersections, and aliases.

## Interfaces vs Types

### Interfaces
- Extensible via declaration merging
- Best for object shapes and class contracts
- Use `extends` for inheritance

```typescript
interface User {
  id: number;
  name: string;
  email?: string;  // optional
}

interface Admin extends User {
  role: "admin";
  permissions: string[];
}

const admin: Admin = {
  id: 1,
  name: "Alice",
  role: "admin",
  permissions: ["read", "write"],
};
```

### Types
- Can represent unions, intersections, primitives, tuples
- Use for aliases and complex compositions

```typescript
type ID = string | number;
type Status = "pending" | "success" | "error";
type UserOrAdmin = User | Admin;

// When to use which:
// - Object shape: interface (or type, both work)
// - Union/intersection: type only
// - Tuple: type only
```

## Generics Intro

Generics let you write reusable code that works with multiple types while preserving type safety.

```typescript
// Generic function
function identity<T>(value: T): T {
  return value;
}
const num = identity(42);      // number
const str = identity("hello"); // string

// Generic interface
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

const userResponse: ApiResponse<User> = {
  data: { id: 1, name: "Alice" },
  status: 200,
  message: "OK",
};

// Generic with constraint
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
getProperty({ name: "Alice", age: 30 }, "name"); // string
```

## Utility Types

Built-in types that transform other types.

### Partial & Required
```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// All properties optional (useful for updates)
type UserUpdate = Partial<User>;
const update: UserUpdate = { name: "Bob" }; // OK

// All properties required (reverse of Partial)
type RequiredUser = Required<Pick<User, "email">>;
```

### Pick & Omit
```typescript
// Pick: subset of fields
type UserPreview = Pick<User, "id" | "name">;
const preview: UserPreview = { id: 1, name: "Alice" };

// Omit: exclude fields
type UserWithoutEmail = Omit<User, "email">;
const userNoEmail: UserWithoutEmail = { id: 1, name: "Alice" };
```

### Record
```typescript
// Record<K, V>: object with keys K and values V
type StatusMap = Record<"pending" | "success" | "error", string>;
const statuses: StatusMap = {
  pending: "Waiting",
  success: "Done",
  error: "Failed",
};

// Dynamic keys
type UserMap = Record<string, User>;
const usersById: UserMap = {
  "1": { id: 1, name: "Alice", email: "a@b.com" },
};
```

### Readonly
```typescript
interface Config {
  apiUrl: string;
}

type ReadonlyConfig = Readonly<Config>;
const config: ReadonlyConfig = { apiUrl: "/api" };
// config.apiUrl = "x"; // Error: Cannot assign to 'apiUrl'
```

## Union Types

A value can be one of several types.

```typescript
type ID = string | number;
type Status = "idle" | "loading" | "success" | "error";

function formatId(id: ID): string {
  return typeof id === "number" ? id.toString() : id;
}

// Union of object types
type Result = 
  | { success: true; data: User }
  | { success: false; error: string };

function handleResult(r: Result) {
  if (r.success) {
    console.log(r.data.name); // narrowed to success branch
  } else {
    console.log(r.error);     // narrowed to error branch
  }
}
```

## Intersection Types

Combine multiple types into one (all properties must be present).

```typescript
interface Named {
  name: string;
}
interface Aged {
  age: number;
}

type Person = Named & Aged;
const person: Person = { name: "Alice", age: 30 };

// With React: extend component props
type ButtonProps = React.ButtonHTMLAttributes<HTMLButtonElement> & {
  variant?: "primary" | "secondary";
};
```

## Type Narrowing

TypeScript narrows types based on control flow.

```typescript
function process(value: string | number) {
  if (typeof value === "string") {
    return value.toUpperCase(); // value is string
  }
  return value.toFixed(2);      // value is number
}

// Truthiness narrowing
function print(str: string | null | undefined) {
  if (str) {
    console.log(str.toUpperCase()); // str is string
  }
}

// Array check
function first(arr: string[] | null): string | undefined {
  if (arr && arr.length > 0) {
    return arr[0]; // arr is string[]
  }
  return undefined;
}
```

## Type Guards

Custom functions that narrow types. Return type is `value is Type`.

```typescript
interface Cat {
  type: "cat";
  meow: () => void;
}
interface Dog {
  type: "dog";
  bark: () => void;
}
type Pet = Cat | Dog;

function isCat(pet: Pet): pet is Cat {
  return pet.type === "cat";
}

function play(pet: Pet) {
  if (isCat(pet)) {
    pet.meow(); // pet is Cat
  } else {
    pet.bark(); // pet is Dog
  }
}

// in operator
function describe(pet: Pet) {
  if ("meow" in pet) {
    pet.meow(); // narrowed
  }
}
```
