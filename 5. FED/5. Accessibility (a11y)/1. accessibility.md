# 1. accessibility

[[5. FED/1. HTML/Semantic HTML]] · [[5. FED/1. HTML/ARIA attributes]]

## Зачем нужна

**Accessibility (a11y)** — доступность интерфейса для людей с ограниченными возможностями: скринридеры, клавиатура, увеличение, ограниченная моторика, когнитивные особенности. Это не «дополнение», а требование для инклюзивного веба.

---

## WAI-ARIA

**WAI-ARIA** (Web Accessibility Initiative — Accessible Rich Internet Applications) — спецификация атрибутов, дополняющих HTML для assistive technologies. [[5. FED/1. HTML/ARIA attributes]]

### Правило: сначала семантика

Нативный HTML предпочтительнее ARIA. `<button>` уже имеет роль, фокус, клавиатуру. ARIA — когда нет подходящего элемента (кастомный виджет, динамический контент).

### Основное

| Категория | Примеры |
| :-- | :-- |
| **Роли** | `role="button"`, `role="dialog"`, `role="alert"` |
| **Состояния** | `aria-expanded`, `aria-pressed`, `aria-checked` |
| **Подписи** | `aria-label`, `aria-labelledby`, `aria-describedby` |
| **Живые регионы** | `aria-live="polite"` для динамического контента |

```html
<!-- Кастомная кнопка без видимого текста -->
<div role="button" tabindex="0" aria-label="Закрыть" onclick="close()">×</div>

<!-- Лучше: нативный элемент -->
<button type="button" aria-label="Закрыть" onclick="close()">×</button>
```

## Semantic HTML

Элементы по назначению дают структуру для скринридеров и навигации. [[5. FED/1. HTML/Semantic HTML]]

### Структура страницы

```html
<header>
  <nav aria-label="Главное меню">...</nav>
</header>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>
```

### Ключевые элементы

- `<main>` — главный контент (один на страницу)
- `<nav>` — навигация
- `<article>` — самостоятельный контент
- `<section>` — тематическая группа (с заголовком)
- `<button>` для действий, `<a href>` для переходов

### Формы

```html
<label for="email">Email</label>
<input type="email" id="email" required aria-describedby="email-hint">
<span id="email-hint">Мы не передаём email третьим лицам</span>

<div role="alert" aria-live="assertive">
  Ошибка: неверный формат
</div>
```

## Keyboard navigation

### Tab order

- Все интерактивные элементы должны быть доступны по Tab
- Порядок — `tabindex` (0 — в потоке, 1+ — в начале, -1 — фокусируемый программно, но не в tab order)

```html
<!-- Плохо: div не в tab order -->
<div onclick="submit()">Отправить</div>

<!-- Хорошо -->
<button type="submit">Отправить</button>
```

### Focus management

- Видимое кольцо фокуса (`:focus-visible`) — не убирать без замены
- При открытии модалки — фокус внутрь, при закрытии — возврат на триггер
- При смене страницы/секции — фокус на заголовок или начало контента

```javascript
// Модалка: при открытии — фокус в диалог
dialog.showModal();
dialog.querySelector('button').focus();

// При закрытии — возврат
const trigger = document.activeElement;
dialog.addEventListener('close', () => trigger?.focus());
```

### Клавиатурные паттерны

| Компонент | Клавиши |
| :-- | :-- |
| Кнопка | Enter, Space |
| Ссылка | Enter |
| Модалка | Escape — закрыть, Tab — ловушка внутри |
| Меню/аккордеон | Arrow Up/Down, Enter, Escape |
| Tab | Arrow Left/Right |

```javascript
// Обработка Enter/Space для кнопки на div
element.addEventListener('keydown', (e) => {
  if (e.key === 'Enter' || e.key === ' ') {
    e.preventDefault();
    element.click();
  }
});
```

### Skip links

Ссылка «Перейти к контенту» в начале страницы — для пользователей клавиатуры.

```html
<a href="#main" class="skip-link">Перейти к контенту</a>
<!-- ... header, nav ... -->
<main id="main">...</main>
```

```css
.skip-link {
  position: absolute;
  left: -9999px;
  top: 0;
}
.skip-link:focus {
  left: 0;
  z-index: 9999;
}
```

## Inclusive web experiences

### Цвет и контраст

- Минимум 4.5:1 для текста (крупный текст — 3:1)
- Не полагаться только на цвет (ошибки — иконка, текст)
- Состояния: не только цвет (подчёркивание, иконка)

### Масштабирование

- `rem`/`em` вместо фиксированных `px` для текста
- Не отключать `zoom` (`user-scalable=no`)

### Движение

- `prefers-reduced-motion: reduce` — отключать или упрощать анимации

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Изображения

- `alt` — осмысленное описание
- Декоративные: `alt=""` или `role="presentation"`
- Сложные изображения — описание в тексте рядом

### Фокус

- Не скрывать `:focus` без замены
- `:focus-visible` — показывать только при навигации с клавиатуры

```css
:focus { outline: 2px solid blue; }
:focus:not(:focus-visible) { outline: none; }  /* для мыши */
:focus-visible { outline: 2px solid blue; }
```

### Тестирование

- Проверка только клавиатурой (Tab, Enter, Escape)
- Скринридер (NVDA, VoiceOver, JAWS)
- Lighthouse (Accessibility audit)
- axe DevTools
