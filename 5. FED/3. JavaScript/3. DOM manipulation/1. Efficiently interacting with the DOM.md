# 1. Efficiently interacting with the DOM

[[5. FED/3. JavaScript/3. DOM manipulation/2. event handling]] · [[5. FED/3. JavaScript/3. DOM manipulation/3. event delegation]]

## Зачем оптимизировать

DOM-операции дорогие: reflow, repaint. Частые чтения/записи тормозят интерфейс. Важно минимизировать обращения и батчить изменения.

## Правила

### 1. Кэшировать ссылки

```javascript
// Плохо — каждый раз поиск в DOM
for (let i = 0; i < 100; i++) {
  document.getElementById('list').appendChild(item);
}

// Хорошо
const list = document.getElementById('list');
for (let i = 0; i < 100; i++) {
  list.appendChild(item);
}
```

### 2. Батчить чтение и запись

Браузер оптимизирует при разделении: сначала все read, потом все write.

```javascript
// Плохо — чередование read/write вызывает лишние reflow
for (const el of elements) {
  el.style.width = el.offsetWidth + 10 + 'px';
}

// Хорошо — read batch, затем write batch
const widths = elements.map(el => el.offsetWidth);
elements.forEach((el, i) => {
  el.style.width = widths[i] + 10 + 'px';
});
```

### 3. DocumentFragment для массовой вставки

```javascript
const fragment = document.createDocumentFragment();
items.forEach(item => {
  const li = document.createElement('li');
  li.textContent = item;
  fragment.appendChild(li);
});
list.appendChild(fragment);  // один reflow вместо N
```

### 4. Селекторы

- `getElementById`, `querySelector` — быстрее `querySelectorAll` при одном элементе
- Узкие селекторы (`.list .item`) быстрее широких (`div span`)

## Связь с событиями

При [[5. FED/3. JavaScript/3. DOM manipulation/3. event delegation|event delegation]] один слушатель на родителе — меньше привязок к DOM, меньше памяти.
