# 1. Debouncing, throttling

[[5. FED/3. JavaScript/5. Performance/2. virtual DOM]] · [[5. FED/3. JavaScript/5. Performance/3. critical rendering path]] · [[4. Common/2. Debounce/1. Debounce]] · [[4. Common/2. Debounce/4. Throttle]]

## Зачем нужны

Ограничить частоту вызовов тяжёлых функций при частых событиях (input, scroll, resize). Без этого — лаги, лишние запросы, перегрузка main thread.

## Debounce

**Суть:** выполнить функцию **после** паузы в вызовах. Каждый новый вызов сбрасывает таймер.

### Для чего

- Поиск в реальном времени — не слать запрос при каждом нажатии клавиши
- Автосохранение — сохранять после паузы ввода
- Валидация формы — проверять поле после остановки печати
- Resize — пересчитывать layout после окончания изменения окна

### Пример

```javascript
function debounce(fn, delay) {
  let timeoutId;
  return function (...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => fn.apply(this, args), delay);
  };
}

const handleSearch = debounce((query) => fetch(`/api/search?q=${query}`), 300);
input.addEventListener('input', (e) => handleSearch(e.target.value));
```

Пользователь печатает "react" — запрос уйдёт один раз, через 300 мс после последней буквы.

## Throttle

**Суть:** выполнять функцию **не чаще** чем раз в N мс. Первый вызов сразу, остальные — по таймеру.

### Для чего

- Scroll — обновлять позицию, подгружать контент (infinite scroll)
- Resize — реагировать на изменение окна с ограничением
- Drag — обрабатывать перемещение
- Mousemove — трекинг без лагов

### Пример

```javascript
function throttle(fn, limit) {
  let inThrottle;
  return function (...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}

const handleScroll = throttle(() => {
  console.log(window.scrollY);
}, 100);

window.addEventListener('scroll', handleScroll);
```

При быстром скролле — вызов раз в 100 мс, а не при каждом событии.

---

## Сравнение

| | Debounce | Throttle |
| :-- | :-- | :-- |
| Когда выполняется | После паузы | Не чаще чем раз в N мс |
| Типичные сценарии | Поиск, автосохранение, валидация | Scroll, resize, mousemove, drag |
| Первый вызов | После паузы | Сразу |

### Критично

- **Debounce** — если нужен только финальный результат (поиск, submit). При scroll с debounce функция не вызовется во время прокрутки — только после остановки.
- **Throttle** — если нужна периодическая реакция во время действия (позиция скролла, прогресс drag).
