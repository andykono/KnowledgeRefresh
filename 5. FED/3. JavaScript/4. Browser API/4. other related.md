## Fetch API

**Зачем:** нативная замена XMLHttpRequest. HTTP-запросы, возвращает Promise. Поддержка async/await, стримов, CORS.

### Основной синтаксис

```javascript
const res = await fetch('/api/data', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ key: 'value' })
});

const data = await res.json();
```

### Опции запроса

| Опция | Назначение |
| :-- | :-- |
| `method` | GET, POST, PUT, DELETE и др. |
| `headers` | Объект заголовков |
| `body` | Тело (строка, FormData, Blob) |
| `mode` | `cors`, `no-cors`, `same-origin` |
| `credentials` | `omit`, `same-origin`, `include` (cookies) |
| `signal` | AbortSignal для отмены |

### Обработка ответа

```javascript
if (!res.ok) throw new Error(res.statusText);
const text = await res.text();
const blob = await res.blob();
```

`res.ok` — true при 200–299. При ошибке HTTP (404, 500) fetch не throw — проверять `res.ok`.

### AbortController

```javascript
const controller = new AbortController();
fetch(url, { signal: controller.signal });
controller.abort();  // отменить запрос
```

## History API

**Зачем:** SPA-навигация без перезагрузки страницы. Изменение URL в адресной строке, кнопки назад/вперёд. SEO-friendly URLs.

### Методы

| Метод | Назначение |
| :-- | :-- |
| `pushState(state, title, url)` | Добавить запись в историю, сменить URL |
| `replaceState(state, title, url)` | Заменить текущую запись (без новой в истории) |
| `go(n)`, `back()`, `forward()` | Навигация по истории |

### Пример

```javascript
history.pushState({ page: 1 }, '', '/page1');
history.replaceState({ page: 2 }, '', '/page2');

window.onpopstate = (e) => {
  // Срабатывает при нажатии назад/вперёд
  const state = e.state;
  renderPage(state?.page);
};
```

### Важно

- `popstate` не срабатывает при `pushState`/`replaceState` — только при навигации пользователем
- `title` в большинстве браузеров игнорируется
- Сервер должен отдавать index.html для всех путей (для SPA)

## Intersection Observer

**Зачем:** отслеживание видимости элемента в viewport. Эффективнее, чем scroll-события — callback вызывается асинхронно, не блокирует main thread. Lazy load изображений, infinite scroll, аналитика просмотров.

### Опции

| Опция | Назначение |
| :-- | :-- |
| `root` | Элемент-контейнер (по умолчанию viewport) |
| `rootMargin` | Отступы от root (`'50px'`, `'0 0 -100px 0'`) |
| `threshold` | 0–1 или массив. Доля видимости для срабатывания |

### Свойства entry

| Свойство | Назначение |
| :-- | :-- |
| `isIntersecting` | Элемент пересекает root |
| `intersectionRatio` | Доля видимости (0–1) |
| `boundingClientRect` | Размеры и позиция элемента |
| `target` | Наблюдаемый элемент |

### Пример: lazy load

```javascript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.src = entry.target.dataset.src ?? '';
        observer.unobserve(entry.target);
      }
    });
  },
  { threshold: 0.1, rootMargin: '50px' }
);
document.querySelectorAll('img[data-src]').forEach((img) => observer.observe(img));
```

## Clipboard API

**Зачем:** копирование и вставка текста/данных через буфер обмена. Кнопка «Скопировать», экспорт в clipboard.

### Запись

```javascript
await navigator.clipboard.writeText('текст');

// Изображение
const blob = await fetch(url).then(r => r.blob());
await navigator.clipboard.write([
  new ClipboardItem({ 'image/png': blob })
]);
```

### Чтение

```javascript
const text = await navigator.clipboard.readText();
```

### Важно

- Требует **user gesture** (клик, нажатие) или разрешение в secure context
- HTTPS или localhost
- `navigator.clipboard` может быть `undefined` — проверять

## Notification API

**Зачем:** push-уведомления в браузере. Уведомления о новых сообщениях, событиях — даже когда вкладка неактивна (с Service Worker).

### Запрос разрешения

```javascript
const permission = await Notification.requestPermission();
// 'granted' | 'denied' | 'default'
```

### Показ уведомления

```javascript
if (Notification.permission === 'granted') {
  new Notification('Заголовок', {
    body: 'Текст уведомления',
    icon: '/icon.png',
    tag: 'unique-id',  // замена предыдущего с тем же tag
    requireInteraction: true  // не исчезает автоматически
  });
}
```

### Важно

- Требует разрешение пользователя
- Для уведомлений при закрытой вкладке — Service Worker
- `tag` — одно уведомление на tag, новое заменяет старое

## ResizeObserver

**Зачем:** отслеживание изменения размеров элемента. Альтернатива `window.resize` — срабатывает при изменении конкретного элемента (sidebar, iframe, контейнер).

```javascript
const observer = new ResizeObserver((entries) => {
  entries.forEach((entry) => {
    const { width, height } = entry.contentRect;
    console.log(width, height);
  });
});
observer.observe(element);
observer.disconnect();  // отписаться
```

## MutationObserver

**Зачем:** отслеживание изменений DOM (добавление/удаление узлов, атрибутов). Более эффективен, чем Mutation events (deprecated). Интеграция с внешними библиотеками, отладка, кастомные элементы.

```javascript
const observer = new MutationObserver((mutations) => {
  mutations.forEach((mutation) => {
    if (mutation.type === 'childList') {
      mutation.addedNodes.forEach((node) => { /* ... */ });
    }
    if (mutation.type === 'attributes') {
      console.log(mutation.attributeName, mutation.target.getAttribute(mutation.attributeName));
    }
  });
});

observer.observe(element, {
  childList: true,
  subtree: true,
  attributes: true,
  attributeFilter: ['class', 'data-id']
});

observer.disconnect();
```

### Опции observe

| Опция | Назначение |
| :-- | :-- |
| `childList` | Добавление/удаление дочерних узлов |
| `subtree` | Включая всех потомков |
| `attributes` | Изменение атрибутов |
| `attributeFilter` | Список отслеживаемых атрибутов |
| `characterData` | Изменение текстового содержимого |
