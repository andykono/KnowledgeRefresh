# 1. Local Storage, Session Storage

[[5. FED/3. JavaScript/4. Browser API/4. other related]]

## Зачем нужны

Хранение данных в браузере (ключ-значение, строка). Без запросов на сервер. Local — между сессиями, Session — до закрытия вкладки.

## API

```javascript
// Запись
localStorage.setItem('key', 'value');
sessionStorage.setItem('key', JSON.stringify(obj));

// Чтение
const value = localStorage.getItem('key');
const obj = JSON.parse(sessionStorage.getItem('key') ?? 'null');

// Удаление
localStorage.removeItem('key');
localStorage.clear();

// Размер
localStorage.length;
```

## Различия

| | localStorage | sessionStorage |
| :-- | :-- | :-- |
| Жизненный цикл | до явного удаления | до закрытия вкладки |
| Область | origin (протокол + домен + порт) | вкладка |
| Объём | ~5–10 MB | ~5–10 MB |

## Важно

- Только строки. Объекты — `JSON.stringify`/`JSON.parse`
- Синхронный API — блокирует main thread при больших объёмах
- Не хранить чувствительные данные (пароли, токены) — доступен JS
- Событие `storage` — срабатывает в других вкладках при изменении (не в текущей)
