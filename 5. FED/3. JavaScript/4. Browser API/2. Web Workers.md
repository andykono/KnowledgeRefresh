# 2. Web Workers

[[5. FED/3. JavaScript/1. Core concepts/4. event loop]] · [[5. FED/3. JavaScript/4. Browser API/4. other related]]

## Зачем нужны

Выполнение кода в **отдельном потоке**. Main thread не блокируется — тяжёлые вычисления, парсинг, криптография. UI остаётся отзывчивым.

## Ограничения

- Нет доступа к DOM, `window`, `document`
- Обмен данными только через `postMessage` (копирование, не shared memory по умолчанию)
- Отдельный глобальный контекст

## Использование

```javascript
// main.js
const worker = new Worker('worker.js');

worker.postMessage({ type: 'compute', data: [1, 2, 3] });

worker.onmessage = (e) => {
  console.log(e.data);  // результат
};

worker.onerror = (e) => console.error(e.message);
worker.terminate();  // завершить
```

```javascript
// worker.js
self.onmessage = (e) => {
  const result = e.data.data.reduce((a, b) => a + b, 0);
  self.postMessage(result);
};
```

## SharedWorker

Один воркер на несколько вкладок. Сложнее в настройке, реже используется.

## Связь с Event Loop

Воркер имеет свой event loop. Main thread и worker общаются асинхронно через сообщения. [[5. FED/3. JavaScript/1. Core concepts/4. event loop]]
