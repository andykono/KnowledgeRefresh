# 3. big O

[[5. FED/3. JavaScript/6. Data structures & algorithms/1. common data structures]] · [[5. FED/3. JavaScript/6. Data structures & algorithms/2. algorithms]]

## Зачем нужен

**Big O** — оценка роста времени/памяти при росте данных. Позволяет сравнивать алгоритмы и выбирать подходящий.

## Правила

- **O(1)** — константа. Не зависит от n.
- **O(n)** — линейно. Один проход.
- **O(n²)** — вложенные циклы по n.
- **O(log n)** — деление пополам (binary search).
- **O(n log n)** — эффективные сортировки.

## Примеры

```javascript
// O(1) — доступ по индексу
arr[i];

// O(n) — один цикл
for (let i = 0; i < arr.length; i++) { }

// O(n²) — вложенные циклы
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) { }
}

// O(log n) — binary search
while (left <= right) {
  mid = (left + right) / 2;
  // ...
}
```

## Упрощение

- Отбрасывать константы: O(2n) → O(n)
- Отбрасывать младшие: O(n² + n) → O(n²)
- Сумма: O(n) + O(m) = O(n + m)
- Вложенность: O(n) × O(m) = O(n × m)

## Примеры по операциям

| Операция | Array | Linked List |
| :-- | :-- | :-- |
| Доступ по индексу | O(1) | O(n) |
| Поиск по значению | O(n) | O(n) |
| Вставка в начало | O(n) | O(1) |
| Вставка в конец | O(1) | O(1)* |
| Удаление по индексу | O(n) | O(n)** |

\* с указателем на хвост  
\** O(1) если есть указатель на узел

## Сортировки

| Алгоритм | Время |
| :-- | :-- |
| Quick Sort | O(n log n) среднее, O(n²) худший |
| Merge Sort | O(n log n) |
| Bubble Sort | O(n²) |
