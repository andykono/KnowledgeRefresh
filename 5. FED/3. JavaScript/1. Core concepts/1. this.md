# 1. this

[[5. FED/3. JavaScript/1. Core concepts/2. closures]] · [[5. FED/3. JavaScript/1. Core concepts/3. prototypes]]

## Зачем нужен

`this` — контекст вызова функции. Определяет, к какому объекту функция «привязана» в момент вызова. Важен для методов объектов, обработчиков событий, конструкторов.

## Как определяется

Зависит от **способа вызова**, не от места объявления:

| Вызов | `this` |
| :-- | :-- |
| `obj.method()` | `obj` |
| `fn()` (без объекта) | `undefined` (strict) / `window` (sloppy) |
| `fn.call(ctx, ...args)` | `ctx` |
| `fn.apply(ctx, args)` | `ctx` |
| `new Fn()` | новый объект |
| Стрелочная функция | лексический `this` (из внешней области) |

## Пример

```javascript
const user = {
  name: 'Anna',
  greet() {
    console.log(this.name);  // this = user
  }
};
user.greet();  // "Anna"

const greet = user.greet;
greet();  // undefined (this не user при вызове без контекста)

// Фиксация: bind, стрелочная функция
const boundGreet = user.greet.bind(user);
setTimeout(boundGreet, 100);  // "Anna"
```

## Стрелочные функции

Не имеют своего `this` — берут из внешней области (замыкание). [[5. FED/3. JavaScript/1. Core concepts/2. closures]]

```javascript
const obj = {
  name: 'Test',
  fn: () => console.log(this.name)  // this — из внешней области, не obj
};
```
