# 4. event loop

## Зачем нужен

**Event Loop** — механизм асинхронного выполнения в JavaScript. Один поток, но неблокирующая работа за счёт очередей. Понимание нужно для правильного порядка выполнения, отладки async-кода.

## Очереди

1. **Call Stack** — синхронный код выполняется сразу
2. **Microtasks** — Promise `.then`/`.catch`/`.finally`, `queueMicrotask`, `MutationObserver`
3. **Macrotasks** — `setTimeout`, `setInterval`, I/O, события (click, scroll)

## Порядок

1. Выполнить весь синхронный код (Call Stack)
2. Обработать все microtasks (до конца)
3. Взять одну macrotask
4. Повторить

## Пример

```javascript
console.log(1);

setTimeout(() => console.log(2), 0);

Promise.resolve(3).then(console.log);

console.log(4);

// Вывод: 1 4 3 2
// Синхронно: 1, 4
// Microtasks: 3
// Macrotask: 2
```

## Важно

- `setTimeout(fn, 0)` не «сразу» — ждёт освобождения стека и microtasks
- Все microtasks выполняются до следующей macrotask
