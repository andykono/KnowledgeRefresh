# 3. prototypes

[[5. FED/3. JavaScript/1. Core concepts/1. this]]

## Зачем нужны

**Prototype** — механизм наследования в JavaScript. Свойства и методы на `prototype` доступны всем экземплярам. Основа классов (ES6 `class` — синтаксический сахар над прототипами).

## Цепочка прототипов

```
obj → obj.__proto__ → Object.prototype → null
```

При обращении к `obj.prop` поиск идёт: сам объект → `__proto__` → `__proto__.__proto__` → … → `null`.

## Пример

```javascript
function Person(name) {
  this.name = name;
}
Person.prototype.greet = function () {
  return `Hi, ${this.name}`;
};

const p = new Person('Anna');
p.greet();  // "Hi, Anna" — метод из prototype
p.hasOwnProperty('greet');  // false
p.hasOwnProperty('name');  // true
```

## ES6 class

```javascript
class Person {
  constructor(name) {
    this.name = name;
  }
  greet() {
    return `Hi, ${this.name}`;
  }
}
// greet — на Person.prototype
```

## Важно

- `__proto__` — устаревший доступ; использовать `Object.getPrototypeOf(obj)`, `Object.setPrototypeOf(obj, proto)`
- `new Fn()` создаёт объект с `__proto__ === Fn.prototype`
