# Semantic HTML

[[5. FED/1. HTML/HTML5 features]] · [[5. FED/1. HTML/ARIA attributes]]

## Определение

**Semantic HTML** — использование HTML-элементов по их назначению, а не только для визуального оформления. Элемент описывает смысл контента, а не его внешний вид.

## Зачем нужен

- **Доступность** — скринридеры и вспомогательные технологии понимают структуру страницы (см. [[5. FED/1. HTML/ARIA attributes]])
- **SEO** — поисковики лучше индексируют контент
- **Поддерживаемость** — код читаем и предсказуем

## Основные семантические элементы

| Элемент | Назначение | Роль (implicit) |
| :-- | :-- | :-- |
| `<header>` | Шапка страницы или секции | `banner` |
| `<nav>` | Навигационные ссылки | `navigation` |
| `<main>` | Основной контент страницы (один на страницу) | `main` |
| `<article>` | Самостоятельный контент (пост, карточка) | `article` |
| `<section>` | Тематическая группа контента | `region` |
| `<aside>` | Дополнительный, косвенный контент | `complementary` |
| `<footer>` | Подвал страницы или секции | `contentinfo` |
| `<figure>` + `<figcaption>` | Иллюстрация с подписью | `figure` |
| `<time>` | Дата/время (атрибут `datetime`) | — |

## Правила

1. **`<main>`** — один на страницу, не внутри `<header>`, `<footer>`, `<aside>`, `<nav>`
2. **`<section>`** — только если есть заголовок (`<h1>`–`<h6>`)
3. **`<article>`** — контент, который имеет смысл отдельно (RSS, переиспользование)
4. **`<div>`** — только когда нет подходящего семантического элемента

## Пример структуры страницы

```html
<header>
  <nav aria-label="Главное меню">...</nav>
</header>

<main>
  <article>
    <h1>Заголовок</h1>
    <section>
      <h2>Подзаголовок</h2>
      <p>Контент...</p>
    </section>
  </article>
  <aside>...</aside>
</main>

<footer>...</footer>
```
