# 6. Animations

[[5. FED/2. CSS/5. Responsive design]] · [[5. FED/2. CSS/7. Performance optimization]]

## Зачем нужны

Обратная связь при взаимодействии (hover, focus), плавные переходы между состояниями, привлечение внимания. Улучшают UX, но не должны перегружать.

## transition

**Зачем:** плавный переход при смене свойства (hover, focus, смена класса). Проще всего для простых эффектов.

```css
/* Кнопка: плавное изменение фона при наведении */
.btn {
  background: blue;
  transition: background 0.3s ease;
}
.btn:hover { background: darkblue; }
```

## animation + @keyframes

**Зачем:** повторяющиеся или многошаговые анимации (лоадер, появление, бегущая строка).

```css
@keyframes spin {
  to { transform: rotate(360deg); }
}
.spinner {
  animation: spin 1s linear infinite;
}
```

## Производительность

Анимировать только `transform` и `opacity` — они не вызывают reflow. Избегать `width`, `height`, `top`, `left`. [[5. FED/2. CSS/7. Performance optimization]]

## prefers-reduced-motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after { animation: none !important; transition: none !important; }
}
```
