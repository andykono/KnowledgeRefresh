# 2. Box Model

[[5. FED/2. CSS/1. Layout (Flexbox, Grid)]] · [[5. FED/2. CSS/5. Responsive design]]

## Зачем нужен

Понимание box model нужно, чтобы предсказуемо задавать размеры, отступы и не получать «лишние» пиксели из-за padding/border. Лежит в основе любого layout.

## Структура

```
content → padding → border → margin
```

- **content** — область содержимого (`width`, `height`)
- **padding** — внутренний отступ
- **border** — граница
- **margin** — внешний отступ (схлопывается с соседними)

## box-sizing

| Значение | Ширина блока |
| :-- | :-- |
| `content-box` (по умолчанию) | `width` = только content |
| `border-box` | `width` = content + padding + border |

```css
* { box-sizing: border-box; }

/* Кнопка: width 200px — итоговая ширина 200px, а не 200 + padding + border */
.btn {
  width: 200px;
  padding: 0.5rem 1rem;
  border: 2px solid;
}
```

## margin collapse

Вертикальные `margin` соседних блоков схлопываются — берётся больший. Не схлопывается: flex/grid-потомки, `overflow: hidden`, padding/border у родителя.
