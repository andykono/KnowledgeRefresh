# 3. Specificity

[[5. FED/2. CSS/4. Inheritance]] · [[5. FED/2. CSS/7. Performance optimization]]

## Зачем нужна

Понять, почему одно правило перебивает другое, и как переопределить стили без `!important`. Важно при отладке и при работе с библиотеками (Bootstrap, Tailwind).

## Определение

**Specificity** — вес селектора, определяющий, какое правило применится при конфликте.

## Порядок (от слабого к сильному)

1. Селекторы элементов и псевдоэлементов (`p`, `::before`)
2. Классы, атрибуты, псевдоклассы (`.class`, `[attr]`, `:hover`)
3. ID (`#id`)
4. Inline-стили (`style="..."`)
5. `!important` (перебивает всё, кроме другого `!important`)

## Подсчёт

- (a, b, c): a = inline/ID, b = классы/атрибуты/псевдоклассы, c = элементы
- `#id .class p` → (1, 1, 1)
- `.class` → (0, 1, 0)

## Пример

```css
/* Побеждает .card — выше специфичность */
.card { color: blue; }
p { color: red; }

/* Переопределение библиотеки: увеличить специфичность, не трогая исходный класс */
.card.card { color: green; }
```

## Правила

- При равной специфичности побеждает последнее правило
- Избегать `!important` и избыточных вложенностей
