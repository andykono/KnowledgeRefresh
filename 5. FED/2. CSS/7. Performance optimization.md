## Зачем нужна

Быстрее отрисовка, меньше лагов при скролле и анимациях. Прямо влияет на UX и метрики (LCP, CLS, FID). Особенно важно на слабых устройствах.

## Селекторы

Избегать глубокой вложенности и универсальных селекторов (`*`). Классы быстрее сложных селекторов. [[5. FED/2. CSS/3. Specificity]]

## Layout и repaint

- **Reflow** — пересчёт геометрии (изменение `width`, `margin`, `display`)
- **Repaint** — перерисовка без изменения layout (`color`, `background`, `visibility`)

Минимизировать reflow: батчить чтение/запись DOM, использовать `transform` вместо `left`/`top`.

## Пример: анимация без reflow

```css
/* Плохо: меняет layout */
.slide { left: 100px; transition: left 0.3s; }

/* Хорошо: только compositing */
.slide { transform: translateX(100px); transition: transform 0.3s; }
```

## will-change

Подсказка браузеру о будущих изменениях. Использовать точечно и кратковременно.

```css
.animated { will-change: transform; }
/* снять после анимации */
```

## contain

Изоляция поддерева для оптимизации layout/paint:

```css
.card { contain: layout style paint; }
```
