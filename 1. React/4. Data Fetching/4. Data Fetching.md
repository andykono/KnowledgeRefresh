# Data Fetching

[[../README.md]]

---

Strategies and patterns for fetching, caching, and synchronizing server data in React applications.

## Overview

Data fetching in React has evolved from manual `useEffect` + `fetch` patterns to sophisticated libraries that handle caching, revalidation, deduplication, and optimistic updates. Choose the right approach based on your app's complexity and requirements.

## Approaches Comparison

| Approach | Best For | Caching | Mutations | DevTools | Bundle Size | Learn |
| :-- | :-- | :-- | :-- | :-- | :-- | :-- |
| **Fetch / Axios** | Simple apps, one-off requests | Manual | Manual | None | Minimal | [[4.1 Fetch and Axios Patterns.md]] |
| **TanStack Query** | Complex apps, real-time sync | Built-in, configurable | useMutation, invalidation | Yes | ~13kb | [[4.2 TanStack Query.md]] |
| **SWR** | Vercel stack, lightweight needs | Built-in, stale-while-revalidate | mutate() | Limited | ~4kb | [[4.3 SWR.md]] |
| **Suspense + use()** | React 19+, streaming SSR | Framework-dependent | Varies | React DevTools | Native | [[4.4 Suspense for Data.md]] |

## Quick Decision Guide

- **Simple CRUD, few screens** → Fetch/Axios with custom hooks
- **Dashboard, real-time data, complex cache** → TanStack Query
- **Next.js, Vercel, lightweight** → SWR
- **React 19, RSC, streaming** → Suspense + use()

## Key Concepts Across All Approaches

- **Loading states** - Show skeletons, spinners, or suspense fallbacks
- **Error handling** - User-friendly messages, retry logic
- **TypeScript** - Type your API responses and request payloads
- **AbortController** - Cancel in-flight requests on unmount
