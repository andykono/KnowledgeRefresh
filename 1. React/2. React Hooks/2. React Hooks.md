## Essential Rules

### Rule 1: Top Level Only
Hooks must be called at the top level of a component, not inside:
- ❌ Loops
- ❌ Conditionals
- ❌ Nested functions

```typescript
// ❌ WRONG
function Component() {
  if (condition) {
    const [state, setState] = useState(0);  // Error!
  }
}

// ✅ CORRECT
function Component() {
  const [state, setState] = useState(0);
  if (condition) {
    // Use state here
  }
}
```

### Rule 2: React Functions Only
Hooks can only be called from:
- ✅ React components
- ✅ Custom hooks

```typescript
// ✅ CORRECT
function MyComponent() {
  const [state, setState] = useState(0);
}

function useCustom() {
  const [state, setState] = useState(0);
}

// ❌ WRONG
const state = useState(0);  // Regular function

setTimeout(() => {
  const [state, setState] = useState(0);  // Callback
}, 1000);
```

See: [[2.1 Rules of Hooks.md|Rules of Hooks]] for detailed explanation.

## Hook Usage Patterns

### Simple State
```typescript
const [count, setCount] = useState(0);
```

### Lazy Initialization
```typescript
const [state, setState] = useState(() => expensiveCalculation());
```

### Multiple State Variables
```typescript
const [x, setX] = useState(0);
const [y, setY] = useState(0);
const [isActive, setIsActive] = useState(false);
```

### State with Object
```typescript
const [user, setUser] = useState({ name: '', email: '' });
setUser(prev => ({ ...prev, name: 'John' }));
```

### Conditional Side Effects
```typescript
useEffect(() => {
  if (id) {
    fetchUser(id);
  }
}, [id]);
```

### Combined Hooks
```typescript
function useAsync(fn) {
  const [state, setState] = useState({ loading: true, data: null });
  useEffect(() => {
    fn().then(data => setState({ loading: false, data }));
  }, [fn]);
  return state;
}
```

## Hook Best Practices

✅ **DO:**
- Learn [[2.1 Rules of Hooks.md|Rules of Hooks]] first (critical!)
- Use useState for simple state
- Use useReducer for complex state
- Put useEffect cleanup in return function
- Use dependency arrays correctly
- Create custom hooks for reusable logic
- Use TypeScript for hook types

❌ **DON'T:**
- Don't call hooks inside loops/conditionals
- Don't call hooks from regular functions
- Don't overuse useMemo/useCallback
- Don't forget cleanup function in useEffect
- Don't forget dependency array in useEffect
- Don't use hooks in class components (use React.useState instead)
