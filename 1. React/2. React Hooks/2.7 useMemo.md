# useMemo Hook

[[2. React Hooks.md]]

---

useMemo memoizes an expensive computed value, returning the cached result until dependencies change.

## How useMemo Works Internally

- **Caches value between renders:** React stores the computed value and returns it on subsequent renders without re-running the computation.
- **Compares deps with Object.is:** When any dependency changes (compared via `Object.is`), React discards the cached value and recomputes.
- **Runs during render:** The memoized function executes during the render phase, not in an effect—so it must be pure and side-effect free.

```typescript
// Signature
function useMemo<T>(factory: () => T, deps: DependencyList): T;
```

---

## Basic Usage

```typescript
// ❌ Without useMemo - recalculates every render
function Component({ data }) {
  const filteredData = data.filter(item => item.active);  // Every render!
  return <div>{filteredData.length}</div>;
}

// ✅ With useMemo - cached
function Component({ data }) {
  const filteredData = useMemo(() => {
    return data.filter(item => item.active);
  }, [data]);  // Only recalculate if data changes
  return <div>{filteredData.length}</div>;
}
```

---

## When to Use

### 1. Expensive Calculations

Array filtering, sorting, or complex math that would slow down renders:

```typescript
const sortedItems = useMemo(() => {
  return [...items].sort((a, b) => a.name.localeCompare(b.name));
}, [items]);
```

### 2. Referential Equality for Objects/Arrays Passed as Props

Child components wrapped in `React.memo` re-render when props change by reference. Creating objects/arrays inline creates new references every render:

```typescript
// ❌ New object every render - breaks React.memo
<Child config={{ theme: 'dark' }} />

// ✅ Stable reference
const config = useMemo(() => ({ theme: 'dark' }), []);
<Child config={config} />
```

### 3. Stable Dependency for useEffect

When an object or array is a dependency of `useEffect`, useMemo prevents unnecessary effect runs:

```typescript
const options = useMemo(() => ({ page: 1, limit: 10 }), []);
useEffect(() => {
  fetchData(options);
}, [options]);  // options is stable unless deps change
```

---

## When NOT to Use

- **Simple calculations:** `count * 2`, string concatenation—overhead of useMemo exceeds benefit.
- **Primitive values:** Primitives are compared by value; no referential equality issue.
- **Premature optimization:** Profile first. Memoization adds complexity and memory.

```typescript
// ❌ Unnecessary
const doubled = useMemo(() => count * 2, [count]);

// ✅ Correct - only for expensive operations
const sorted = useMemo(() => {
  return largeArray.sort((a, b) => a.name.localeCompare(b.name));
}, [largeArray]);
```

---

## useMemo vs useCallback

| Hook | Caches | Returns |
|------|--------|---------|
| `useMemo` | Value | The computed value |
| `useCallback` | Function | A memoized function |

```typescript
// useMemo - caches the result of calling the function
const value = useMemo(() => computeExpensiveValue(a, b), [a, b]);

// useCallback - caches the function itself (equivalent to useMemo returning a function)
const fn = useCallback(() => doSomething(a, b), [a, b]);
// Same as: const fn = useMemo(() => () => doSomething(a, b), [a, b]);
```

---

## useMemo with Objects and Arrays

Creating stable references prevents unnecessary re-renders and effect triggers:

```typescript
// Stable empty array for default
const items = useMemo(() => [] as Item[], []);

// Stable config object
const chartOptions = useMemo(() => ({
  responsive: true,
  plugins: { legend: { position: 'top' } }
}), []);

// Derived object from props
const filterState = useMemo(() => ({
  active: status === 'active',
  query: searchQuery
}), [status, searchQuery]);
```

---

## Custom Comparison Isn't Possible

Unlike `React.memo`, which accepts a custom comparison function, `useMemo` always uses `Object.is` for dependencies. You cannot provide a custom equality check. For complex dependency logic, derive a primitive or use a ref.

```typescript
// React.memo allows custom comparison
const MemoChild = React.memo(Child, (prev, next) => prev.id === next.id);

// useMemo does NOT - always Object.is for deps
const value = useMemo(() => compute(x), [x]);  // x compared with Object.is
```

---

## React Compiler and Automatic Memoization

The React Compiler (React 19+) can automatically memoize values and callbacks. In a compiler-enabled project, many manual `useMemo` and `useCallback` calls may become unnecessary. The compiler analyzes dependencies and inserts memoization where beneficial. For now, manual memoization remains the standard approach.

## Common Mistakes

### Forgetting Dependencies

```typescript
// ❌ Missing dep - stale closure
const filtered = useMemo(() => data.filter(d => d.type === filterType), [data]);

// ✅ Include all deps
const filtered = useMemo(() => data.filter(d => d.type === filterType), [data, filterType]);
```

### Using Inside Loops

```typescript
// ❌ Invalid - hooks can't be called conditionally or in loops
items.map(item => {
  const computed = useMemo(() => process(item), [item]);  // WRONG
  return <div>{computed}</div>;
});

// ✅ Memoize at parent or use a wrapper component
```

### Memoizing Cheap Operations

```typescript
// ❌ Overhead > benefit
const label = useMemo(() => `${name} (${count})`, [name, count]);

// ✅ Just compute
const label = `${name} (${count})`;
```

## TypeScript Typing Examples

```typescript
// Generic type inferred from factory return
const value = useMemo(() => ({ id: 1, name: 'test' }), []);
// value: { id: number; name: string }

// Explicit generic when inference fails
const items = useMemo<Item[]>(() => fetchItems(), []);
// items: Item[]

// With complex return type
interface ChartData {
  labels: string[];
  datasets: { data: number[] }[];
}
const chartData = useMemo<ChartData>(() => ({
  labels: ['A', 'B', 'C'],
  datasets: [{ data: [1, 2, 3] }]
}), []);
```
