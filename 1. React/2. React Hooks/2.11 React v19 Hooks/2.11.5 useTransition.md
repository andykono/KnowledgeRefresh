# useTransition Hook

[[2.11 React v19 Hooks.md]]

---

useTransition marks state updates as transitions for non-blocking UI updates.

## Basic Usage

```typescript
'use client'

import { useTransition } from 'react';
import { useState } from 'react';

function SearchResults({ query }) {
  const [isPending, startTransition] = useTransition();
  const [results, setResults] = useState([]);

  const handleSearch = (newQuery) => {
    // Mark as transition (non-blocking)
    startTransition(async () => {
      const searchResults = await search(newQuery);
      setResults(searchResults);
    });
  };

  return (
    <div>
      <input onChange={(e) => handleSearch(e.target.value)} />
      {isPending && <div>Searching...</div>}
      <ul>
        {results.map(result => (
          <li key={result.id}>{result.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

---

## Anatomy

```typescript
const [isPending, startTransition] = useTransition();

// isPending: boolean - Transition is running
// startTransition: (callback) => void - Mark update as transition
```

---

## Key Differences from v18

### In React v18
- useTransition for async operations
- startTransition marks setState calls
- No cancellation

### In React v19
- useTransition works with actions
- Can cancel transitions
- Better server integration

## Common Patterns

### Search Input
```typescript
'use client'

import { useTransition } from 'react';

function Search() {
  const [isPending, startTransition] = useTransition();
  const [results, setResults] = useState([]);
  const [query, setQuery] = useState('');

  const handleSearch = (e) => {
    const value = e.target.value;
    setQuery(value);

    startTransition(async () => {
      const data = await searchAPI(value);
      setResults(data);
    });
  };

  return (
    <div>
      <input value={query} onChange={handleSearch} />
      {isPending && <Spinner />}
      {results.map(r => <div key={r.id}>{r.name}</div>)}
    </div>
  );
}
```

### Slow Component
```typescript
'use client'

import { useTransition } from 'react';

function Dashboard({ data }) {
  const [isPending, startTransition] = useTransition();
  const [view, setView] = useState('list');

  const handleChangeView = (newView) => {
    // Switching views doesn't block UI
    startTransition(() => {
      setView(newView);
    });
  };

  return (
    <div>
      <button onClick={() => handleChangeView('list')}>List</button>
      <button onClick={() => handleChangeView('grid')}>Grid</button>

      {isPending && <div>Loading view...</div>}

      {view === 'list' ? (
        <SlowListView data={data} />
      ) : (
        <SlowGridView data={data} />
      )}
    </div>
  );
}
```

## vs useDeferredValue

| Hook | Use Case |
| :-- | :-- |
| **useTransition** | Mark state update | Mark setState as low-priority |
| **useDeferredValue** | Defer a value | Value from parent prop |

```typescript
// useTransition: Control the update
const [isPending, startTransition] = useTransition();
startTransition(() => setCount(count + 1));

// useDeferredValue: Receive a prop
const deferredValue = useDeferredValue(value);
```

See: [[2.11.6 useDeferredValue.md|useDeferredValue]]

## Best Practices

✅ **DO:**
- Use for expensive re-renders
- Show pending feedback
- Use with actions for best integration
- Combine with useOptimistic

❌ **DON'T:**
- Don't use for every state update
- Don't hide loading states
- Don't use for critical updates
- Don't overuse (can slow down app)
