# useDeferredValue Hook

[[2.11 React v19 Hooks.md]]

---

useDeferredValue defers a value update for non-blocking re-renders.

## Basic Usage

```typescript
'use client'

import { useDeferredValue, useState } from 'react';

function SearchPage() {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search..."
      />

      {/* This re-renders with deferredQuery after input updates */}
      <SearchResults query={deferredQuery} />
    </div>
  );
}

function SearchResults({ query }) {
  const results = useMemo(() => {
    // Expensive computation
    return filteredResults(query);
  }, [query]);

  return (
    <ul>
      {results.map(r => (
        <li key={r.id}>{r.name}</li>
      ))}
    </ul>
  );
}
```

---

## How It Works

```
Input value changes → User sees input update immediately
                    ↓ (Later)
                    ↓ Deferred value updates component
                    → Results update with a small delay
```

---

## vs useTransition

| Hook | Who Controls | Usage |
| :-- | :-- | :-- |
| **useTransition** | You (startTransition) | Mark state update |
| **useDeferredValue** | React (automatic) | Receive prop value |

```typescript
// useTransition: You control it
const [isPending, startTransition] = useTransition();
startTransition(() => setQuery(value));

// useDeferredValue: Automatic
const deferredQuery = useDeferredValue(query);
```

See: [[2.11.5 useTransition.md|useTransition]]

## Common Patterns

### Search with Expensive Filter
```typescript
'use client'

import { useDeferredValue, useMemo } from 'react';

export function SearchList({ items }) {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);

  const filteredItems = useMemo(() => {
    // This might be slow with 10,000+ items
    return items.filter(item =>
      item.name.toLowerCase().includes(deferredQuery.toLowerCase())
    );
  }, [deferredQuery, items]);

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search..."
      />

      <ul>
        {filteredItems.length === 0 ? (
          <p>No results for "{query}"</p>
        ) : (
          filteredItems.map(item => (
            <li key={item.id}>{item.name}</li>
          ))
        )}
      </ul>
    </div>
  );
}
```

### Data with Suspense
```typescript
'use client'

import { useDeferredValue, Suspense } from 'react';

function TabContainer() {
  const [tab, setTab] = useState('home');
  const deferredTab = useDeferredValue(tab);

  return (
    <div>
      <button onClick={() => setTab('home')}>Home</button>
      <button onClick={() => setTab('profile')}>Profile</button>

      {/* User sees button change immediately */}
      {/* Content updates with deferredTab (with Suspense) */}
      <Suspense fallback={<Loading />}>
        <TabContent tab={deferredTab} />
      </Suspense>
    </div>
  );
}
```

## Combined with Other Hooks

```typescript
'use client'

import { useDeferredValue, useTransition, useMemo } from 'react';

function AdvancedSearch({ data }) {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);

  // Also use useTransition if needed
  const [isPending, startTransition] = useTransition();

  const results = useMemo(() => {
    return expensiveSearch(deferredQuery, data);
  }, [deferredQuery, data]);

  return (
    <div>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      {isPending && <div>Searching...</div>}
      <ResultsList results={results} />
    </div>
  );
}
```

## Best Practices

✅ **DO:**
- Use with expensive filters/searches
- Combine with useMemo for computations
- Show feedback (isPending indicator)
- Use with Suspense if applicable

❌ **DON'T:**
- Don't use for every prop
- Don't use without expensive operations
- Don't ignore user feedback
- Don't use when synchronous is better
