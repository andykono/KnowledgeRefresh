React v19 introduces powerful new hooks for server integration, performance, and form handling.

## React v19 Hooks Overview

| Hook | Purpose | When | Learn |
| :-- | :-- | :-- | :-- |
| **use()** | Unwrap promises and context | Conditional unwrapping | [[2.11.1 use().md]] |
| **useActionState** | Server form actions | Form submission | [[2.11.2 useActionState.md]] |
| **useFormStatus** | Get form submission status | Show pending state | [[2.11.3 useFormStatus.md]] |
| **useOptimistic** | Optimistic UI updates | Update before server | [[2.11.4 useOptimistic.md]] |
| **useTransition** | Non-blocking state updates | Mark as transition | [[2.11.5 useTransition.md]] |
| **useDeferredValue** | Defer value updates | Heavy computations | [[2.11.6 useDeferredValue.md]] |
| **useLayoutEffect** | Before browser paint | DOM measurements | [[2.11.7 useLayoutEffect & useInsertionEffect.md]] |

## What's New in v19?

### Server Functions
```typescript
// Server function
'use server'

export async function submitForm(formData: FormData) {
  const email = formData.get('email');
  await db.users.create({ email });
  return { success: true };
}
```

### Form Actions
```typescript
'use client'

import { submitForm } from './actions';

export function Form() {
  const [state, formAction] = useActionState(submitForm, null);

  return (
    <form action={formAction}>
      <input name="email" type="email" />
      <button type="submit">Submit</button>
      {state?.success && <p>Success!</p>}
    </form>
  );
}
```

### Optimistic Updates
```typescript
function TodoItem({ id, text }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    [{ id, text }],
    (state, newTodo) => [...state, newTodo]
  );

  const handleAdd = async (formData) => {
    const todo = Object.fromEntries(formData);
    addOptimisticTodo(todo);
    await saveTodo(todo);
  };

  return (
    <form action={handleAdd}>
      <input name="text" />
      <button>Add</button>
      <ul>
        {optimisticTodos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </form>
  );
}
```

### Promise Unwrapping
```typescript
'use client'

function Component() {
  // Can await in render!
  const data = use(fetchData());

  return <div>{data}</div>;
}
```

## Migration from v18

### Before (v18)
```typescript
function Form() {
  const [state, setState] = useState(null);
  const [isPending, setIsPending] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault();
    setIsPending(true);
    try {
      const result = await submitForm(new FormData(e.currentTarget));
      setState(result);
    } finally {
      setIsPending(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="email" />
      <button disabled={isPending}>
        {isPending ? 'Submitting...' : 'Submit'}
      </button>
    </form>
  );
}
```

### After (v19)
```typescript
function Form() {
  const [state, formAction] = useActionState(submitForm, null);

  return (
    <form action={formAction}>
      <input name="email" />
      <button type="submit">Submit</button>
    </form>
  );
}
```

Much simpler! React handles loading state automatically.

## Key Concepts

### Actions
Functions that can run on server or client:
- ✅ Can read/write database
- ✅ Can run secret code
- ✅ Can revalidate cache
- ❌ Cannot be used in regular event handlers (must be in form/button)

### Transitions
Mark state updates as transitions for better UX:
- ✅ Non-blocking (UI stays responsive)
- ✅ Can show pending state
- ✅ Can cancel in v19

### Optimistic Updates
Show data before server confirms:
- ✅ Instant user feedback
- ✅ Rollback if server rejects
- ✅ Better perceived performance

## React v19 Best Practices

✅ **DO:**
- Use Server Functions for database access
- Use useActionState for forms (not useState)
- Use useOptimistic for instant feedback
- Use useFormStatus in forms
- Use Suspense with use()
- Mark server code with 'use server'

❌ **DON'T:**
- Don't use form actions in onclick (use form submission)
- Don't fetch in every render (use actions)
- Don't forget 'use server' directive
- Don't ignore security (server validation)
- Don't overuse useTransition (for complex updates)

## Quick Examples

### Simple Form
```typescript
'use client'

async function submitForm(formData) {
  'use server'
  const email = formData.get('email');
  // Send to database
  return { success: true };
}

export function SignupForm() {
  const [state, formAction] = useActionState(submitForm, null);

  return (
    <form action={formAction}>
      <input name="email" type="email" required />
      <button type="submit">Sign Up</button>
      {state?.success && <p>✓ Signed up!</p>}
    </form>
  );
}
```

### Todo with Optimistic
```typescript
'use client'

async function addTodo(formData) {
  'use server'
  const text = formData.get('text');
  const todo = { id: crypto.randomUUID(), text, done: false };
  await db.todos.create(todo);
  return todo;
}

export function TodoList({ initialTodos }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    initialTodos,
    (state, newTodo) => [...state, newTodo]
  );

  const [state, formAction] = useActionState(addTodo, null);

  const handleSubmit = (formData) => {
    const text = formData.get('text');
    addOptimisticTodo({ id: '...', text, done: false });
    formAction(formData);
  };

  return (
    <form action={handleSubmit}>
      <input name="text" required />
      <button>Add Todo</button>
      <ul>
        {optimisticTodos.map(todo => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </form>
  );
}
```
