# useLayoutEffect & useInsertionEffect

[[2.11 React v19 Hooks.md]]

---

Two specialized hooks for effects that run before or after the browser paints.

## useLayoutEffect

Runs **after DOM mutations** but **before browser paint**.

```typescript
'use client'

import { useLayoutEffect, useRef, useState } from 'react';

function Tooltip({ show }) {
  const ref = useRef(null);
  const [position, setPosition] = useState(null);

  useLayoutEffect(() => {
    if (!ref.current) return;

    // Measure DOM after changes but before paint
    const rect = ref.current.getBoundingClientRect();
    setPosition({
      top: rect.top,
      left: rect.left,
    });
  }, [show]);

  return (
    <div ref={ref}>
      {show && position && (
        <div style={{ position: 'absolute', top: position.top }}>
          Tooltip
        </div>
      )}
    </div>
  );
}
```

---

## useInsertionEffect

Runs **before layout effects**, used by CSS-in-JS libraries.

```typescript
'use client'

import { useInsertionEffect } from 'react';

function Component() {
  useInsertionEffect(() => {
    // Inject styles before any layout effects
    const style = document.createElement('style');
    style.textContent = `
      .component {
        color: red;
      }
    `;
    document.head.appendChild(style);

    return () => {
      style.remove();
    };
  }, []);

  return <div className="component">Content</div>;
}
```

## Timing Comparison

```
Render Phase
    ↓
Commit Phase
    ↓
useInsertionEffect ← Runs here (rare, libraries only)
    ↓
Layout Calculations
    ↓
useLayoutEffect ← Runs here (for measurements)
    ↓
Browser Paints
    ↓
useEffect ← Runs here (most code)
```

## useLayoutEffect Use Cases

### Measure & Adjust Layout
```typescript
'use client'

import { useLayoutEffect, useRef, useState } from 'react';

function FlexibleBox() {
  const ref = useRef(null);
  const [height, setHeight] = useState(null);

  useLayoutEffect(() => {
    if (ref.current) {
      // Get height before paint
      setHeight(ref.current.offsetHeight);
    }
  }, []);

  return <div ref={ref}>Content: {height}px</div>;
}
```

### Scroll Position
```typescript
'use client'

import { useLayoutEffect, useRef } from 'react';

function ScrollToBottom() {
  const ref = useRef<HTMLDivElement>(null);

  useLayoutEffect(() => {
    if (ref.current) {
      // Scroll before paint - no flicker
      ref.current.scrollTop = ref.current.scrollHeight;
    }
  }, []);

  return <div ref={ref} style={{ height: '300px', overflow: 'auto' }}>
    Content
  </div>;
}
```

## Common Mistakes

### ❌ Using useLayoutEffect Everywhere
```typescript
// WRONG: useLayoutEffect blocks paint
useLayoutEffect(() => {
  trackPageView();  // This doesn't need layout!
}, []);

// CORRECT: Use useEffect
useEffect(() => {
  trackPageView();
}, []);
```

### ❌ Forgetting About Performance
```typescript
// WRONG: Blocks paint while measuring
useLayoutEffect(() => {
  while (expensive operation) {
    // ...
  }
}, []);

// CORRECT: Move expensive work to useEffect
useLayoutEffect(() => {
  const measurement = element.offsetHeight;
  return () => {
    // Cleanup
  };
}, []);

useEffect(() => {
  // Expensive work after paint
  processLargeData(measurement);
}, [measurement]);
```

## When to Use Each

### useEffect (99% of code)
```typescript
// ✅ API calls
useEffect(() => {
  fetchData();
}, []);

// ✅ Event listeners
useEffect(() => {
  window.addEventListener('resize', handler);
  return () => window.removeEventListener('resize', handler);
}, []);

// ✅ Analytics/logging
useEffect(() => {
  trackEvent('page_viewed');
}, []);
```

### useLayoutEffect (1% of code)
```typescript
// ✅ Measure DOM
useLayoutEffect(() => {
  const height = ref.current.offsetHeight;
  setHeight(height);
}, []);

// ✅ Adjust scroll position
useLayoutEffect(() => {
  ref.current.scrollTop = ref.current.scrollHeight;
}, []);

// ✅ DOM mutations before paint
useLayoutEffect(() => {
  ref.current.style.display = 'none';
}, []);
```

### useInsertionEffect (Rare - Library Code)
```typescript
// ✅ CSS-in-JS libraries only
useInsertionEffect(() => {
  const style = createStyleElement();
  document.head.appendChild(style);
  return () => style.remove();
}, []);
```

---

## Summary

| Hook | When Runs | Use Case |
| :-- | :-- | :-- |
| **useEffect** | After paint (async) | Most effects (99%) |
| **useLayoutEffect** | Before paint (sync) | DOM measurements (1%) |
| **useInsertionEffect** | Before layout (sync) | CSS-in-JS only |

---

## Best Practices

✅ **DO:**
- Default to useEffect
- Only use useLayoutEffect if measuring
- Use useInsertionEffect only in libraries
- Be careful with performance

❌ **DON'T:**
- Don't use useLayoutEffect by default
- Don't put expensive work in useLayoutEffect
- Don't use useInsertionEffect in regular code
- Don't forget cleanup functions
