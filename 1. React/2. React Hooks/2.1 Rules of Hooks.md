# Rules of Hooks

[[2. React Hooks.md]]

---

**These rules are CRITICAL.** Breaking them will cause bugs. React has a linter to check these rules.

## The Two Rules

### Rule 1: Only Call Hooks at the Top Level

Hooks must be called at the top level of your component or custom hook. **Never** call them inside:
- ❌ Loops
- ❌ Conditionals
- ❌ Nested functions

```typescript
// ❌ WRONG: Inside condition
function Component() {
  if (condition) {
    const [state, setState] = useState(0);  // ERROR!
  }
}

// ❌ WRONG: Inside loop
function Component() {
  for (let i = 0; i < 10; i++) {
    const [state, setState] = useState(0);  // ERROR!
  }
}

// ❌ WRONG: Inside callback
function Component() {
  const handleClick = () => {
    const [state, setState] = useState(0);  // ERROR!
  };
}

// ✅ CORRECT: Top level only
function Component() {
  const [state, setState] = useState(0);
  const [count, setCount] = useState(0);

  if (condition) {
    // Use state here - fine
  }

  const handleClick = () => {
    setState(state + 1);  // Use state here - fine
  };

  return <div>{state}</div>;
}
```

### WHY This Rule?

React keeps track of hooks in a specific order based on function calls:

```typescript
function Component() {
  const [state1, setState1] = useState(0);   // Hook #1
  const [state2, setState2] = useState('');  // Hook #2
  const [state3, setState3] = useState(false); // Hook #3

  useEffect(() => {
    // ...
  }, []); // Hook #4
}
```

React internally maps:
```
Component → [
  { state: 0, setState: ... },
  { state: '', setState: ... },
  { state: false, setState: ... },
  { effect: ..., deps: [] }
]
```

If you call hooks conditionally, the order changes:

```typescript
// ❌ DON'T DO THIS
function Component({ user }) {
  if (user) {
    const [state, setState] = useState('');  // Hook #1 or #2?
  }
  useEffect(() => {
    // ...
  }, []); // Hook #1 or #2?
}

// On first render:
// useState (Hook #1)
// useEffect (Hook #2)

// If user becomes null on re-render:
// useEffect (Hook #1)
// But React expects it at Hook #2!
// This causes: "Rendered fewer hooks than expected" ERROR
```

### Rule 2: Only Call Hooks from React Functions

Hooks can only be called from:
- ✅ React components
- ✅ Custom hooks

```typescript
// ✅ CORRECT: In a React component
function MyComponent() {
  const [state, setState] = useState(0);
  return <div>{state}</div>;
}

// ✅ CORRECT: In a custom hook
function useCustom() {
  const [state, setState] = useState(0);
  return state;
}

// ❌ WRONG: In a regular JavaScript function
function regularFunction() {
  const [state, setState] = useState(0);  // ERROR!
}

// ❌ WRONG: In a callback
setTimeout(() => {
  const [state, setState] = useState(0);  // ERROR!
}, 1000);

// ❌ WRONG: In an event listener
document.addEventListener('click', () => {
  const [state, setState] = useState(0);  // ERROR!
});

// ❌ WRONG: In promise.then
fetch('/api').then(() => {
  const [state, setState] = useState(0);  // ERROR!
});
```

## Using Hooks Correctly

### Pattern 1: Conditional State Usage
```typescript
// ✅ CORRECT: Hook at top, use conditionally
function Component({ user }) {
  const [profile, setProfile] = useState(null);

  if (!user) {
    return <LoginForm />;  // Return early if no user
  }

  return <Profile profile={profile} />;
}

// ✅ ALSO CORRECT: Extract into separate component
function ProfileContainer({ user }) {
  if (!user) {
    return <LoginForm />;
  }
  return <Profile user={user} />;
}

function Profile({ user }) {
  const [profile, setProfile] = useState(null);
  // ...
}
```

### Pattern 2: Conditional Effects
```typescript
// ✅ CORRECT: Effect at top, condition inside
function Component({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    if (!userId) return;  // Conditional inside effect
    fetchUser(userId).then(setUser);
  }, [userId]);

  return <div>{user?.name}</div>;
}
```

### Pattern 3: Custom Hooks
```typescript
// ✅ CORRECT: Extract into custom hook
function useUser(userId) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    if (!userId) return;
    fetchUser(userId).then(setUser);
  }, [userId]);

  return user;
}

// Use in component
function Component({ userId }) {
  const user = useUser(userId);
  return <div>{user?.name}</div>;
}
```

## Common Mistakes

### Mistake 1: Conditional Hook Call
```typescript
// ❌ WRONG
function Component({ condition }) {
  if (condition) {
    const [state, setState] = useState(0);
  }
  useEffect(() => {}, []);  // May be hook #1 or #2
}

// ✅ CORRECT
function Component({ condition }) {
  const [state, setState] = useState(condition ? 0 : null);
  useEffect(() => {}, []);
}
```

### Mistake 2: Hook in Callback
```typescript
// ❌ WRONG
function Component() {
  const handleClick = () => {
    const [state, setState] = useState(0);  // Don't!
  };
}

// ✅ CORRECT
function Component() {
  const [state, setState] = useState(0);
  const handleClick = () => {
    setState(state + 1);  // Use state here
  };
}
```

### Mistake 3: Hook in Loop
```typescript
// ❌ WRONG
function Component({ items }) {
  items.forEach(item => {
    const [state, setState] = useState(0);  // Don't!
  });
}

// ✅ CORRECT
function Component({ items }) {
  const [states, setStates] = useState(items.map(() => 0));
}
```

### Mistake 4: Missing Dependencies
```typescript
// ❌ WRONG: Missing dependency
useEffect(() => {
  fetchUser(userId);
}, []);  // userId not included!

// ✅ CORRECT: All dependencies included
useEffect(() => {
  fetchUser(userId);
}, [userId]);
```

## Why These Rules Exist

React's Hook system relies on:
1. **Call order consistency** - Hooks are identified by their order
2. **Function scope** - Hooks need access to React's component state
3. **Dependency tracking** - Effects need to know what changed

These rules ensure:
- ✅ Hooks always run in the same order
- ✅ React can track which hook belongs to which state
- ✅ Closure issues are avoided
- ✅ Dependencies are properly managed

---

## Summary

### The Golden Rules
1. **Only call at top level** - Not in loops, conditions, or nested functions
2. **Only in React functions** - Components or custom hooks

### Check with ESLint
Use `eslint-plugin-react-hooks` to automatically catch violations.

### When Confused
Extract logic into:
- ✅ A custom hook
- ✅ A separate component
- ✅ A utility function
