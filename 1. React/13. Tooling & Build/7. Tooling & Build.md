Overview of the modern React tooling ecosystem: bundlers, frameworks, linting, and deployment.

## Overview

| Category | Tools | Purpose |
| :-- | :-- | :-- |
| **Bundler** | Vite, webpack, Turbopack | Build, dev server, HMR |
| **Framework** | Next.js, Remix | Full-stack, routing, SSR |
| **Linting** | ESLint, Biome | Code quality, consistency |
| **Formatting** | Prettier | Consistent style |
| **CI/CD** | GitHub Actions | Lint, test, deploy |
| **Hosting** | Vercel, Netlify | Zero-config deployment |

---

## Decision Tree

```
Starting a new React project?
│
├─ Need SSR/SSG, file-based routing?
│  └─ Next.js
│
├─ SPA only, fast dev experience?
│  └─ Vite + React
│
├─ Migrating from CRA?
│  └─ Vite (easiest migration path)
│
└─ Enterprise, custom build?
   └─ webpack or Turbopack
```

## Modern Tooling Stack (2024+)

**Recommended for most projects:**
- **Vite** — Fast HMR, simple config, great DX
- **TypeScript** — Type safety
- **ESLint (flat config)** — Linting
- **Prettier** — Formatting
- **Vitest** — Unit testing (optional)
- **Vercel/Netlify** — Deployment

**When to choose Next.js instead:**
- Need server-side rendering
- Need API routes
- Want file-based routing
- Deploying to Vercel (native integration)

## Key Concepts

### HMR (Hot Module Replacement)
Replace modules in the browser without full reload. Vite and webpack support this.

### Tree Shaking
Remove unused exports from the bundle. Requires ES modules.

### Code Splitting
See [[6.3 Code Splitting.md|Code Splitting]] — lazy loading, dynamic imports.

### Path Aliases
`@/components` instead of `../../../components` — configured in tsconfig and bundler.
