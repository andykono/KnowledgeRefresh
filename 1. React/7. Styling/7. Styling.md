# Styling

[[../README.md]]

---

Styling in React offers multiple approaches, each with trade-offs in developer experience, performance, and maintainability. Choose based on your project's scale, team preferences, and design system requirements.

## Overview

React components need styling for layout, typography, theming, and responsive design. Unlike traditional CSS, React styling solutions often provide scoping, composition, and dynamic styling capabilities that integrate with component architecture.

## Approaches Comparison

| Approach | Scoping | Dynamic Styles | TypeScript | Bundle Size | Learning Curve | Learn |
| :-- | :-- | :-- | :-- | :-- | :-- | :-- |
| **CSS Modules** | Automatic (hashed) | Via classes | Full support | Minimal | Low | [[7.1 CSS Modules.md]] |
| **Tailwind CSS** | Utility-based | Via classes | Full support | Tree-shaken | Medium | [[7.2 Tailwind CSS.md]] |
| **CSS-in-JS** | Component-scoped | Runtime props | Full support | 10–40kb | Medium–High | [[7.3 CSS-in-JS.md]] |

## Quick Decision Guide

- **Existing CSS skills, minimal runtime cost** → CSS Modules
- **Rapid prototyping, design system, utility-first** → Tailwind CSS
- **Dynamic theming, component libraries, design tokens** → CSS-in-JS (styled-components, Emotion)

## Key Considerations

| Factor | CSS Modules | Tailwind | CSS-in-JS |
| :-- | :-- | :-- | :-- |
| **Build-time vs runtime** | Build-time | Build-time | Runtime (or compile-time) |
| **SSR support** | Native | Native | Requires setup |
| **Theme switching** | CSS variables | Config + classes | Props-based |
| **Component composition** | compose() | cn() utility | Styled components |
| **IDE support** | Good | Excellent | Good |

## Learning Path

1. [[7.1 CSS Modules.md]] - Master scoped CSS with TypeScript
2. [[7.2 Tailwind CSS.md]] - Utility-first workflow
3. [[7.3 CSS-in-JS.md]] - Dynamic styling with styled-components/Emotion

## Best Practices Across All Approaches

- **Consistency** - Stick to one primary approach per project
- **Design tokens** - Use CSS variables or Tailwind config for colors, spacing
- **Responsive design** - Mobile-first breakpoints
- **Accessibility** - Sufficient contrast, focus indicators
