# CSS-in-JS

[[7. Styling.md]]

---

CSS-in-JS writes styles directly in JavaScript/TypeScript. Offers dynamic styling, scoping, and co-location with components.

## styled-components

### Setup

```bash
npm install styled-components
npm install -D @types/styled-components
```

### Basic Usage

```typescript
import styled from 'styled-components';

const Button = styled.button`
  background: #3b82f6;
  color: white;
  padding: 8px 16px;
  border: none;
  border-radius: 6px;
  cursor: pointer;

  &:hover {
    background: #2563eb;
  }
`;

function App() {
  return <Button>Click me</Button>;
}
```

### Dynamic Props

```typescript
interface ButtonProps {
  $variant?: 'primary' | 'secondary' | 'danger';
  $size?: 'sm' | 'md' | 'lg';
}

const Button = styled.button<ButtonProps>`
  padding: ${({ $size }) =>
    $size === 'sm' ? '4px 8px' :
    $size === 'lg' ? '12px 24px' : '8px 16px'};
  background: ${({ $variant }) =>
    $variant === 'danger' ? '#ef4444' :
    $variant === 'secondary' ? '#6b7280' : '#3b82f6'};
  color: white;
  border: none;
  border-radius: 6px;
`;

// Transient props ($prefix) aren't passed to the DOM
<Button $variant="danger" $size="lg">Delete</Button>
```

### Extending Styles

```typescript
const BaseButton = styled.button`
  padding: 8px 16px;
  border: none;
  border-radius: 6px;
`;

const PrimaryButton = styled(BaseButton)`
  background: #3b82f6;
  color: white;
`;

const OutlineButton = styled(BaseButton)`
  background: transparent;
  border: 2px solid #3b82f6;
  color: #3b82f6;
`;
```

### Global Styles

```typescript
import { createGlobalStyle } from 'styled-components';

const GlobalStyle = createGlobalStyle`
  *, *::before, *::after {
    box-sizing: border-box;
    margin: 0;
  }

  body {
    font-family: 'Inter', sans-serif;
    line-height: 1.6;
  }
`;

function App() {
  return (
    <>
      <GlobalStyle />
      <Main />
    </>
  );
}
```

### Theming

```typescript
import { ThemeProvider } from 'styled-components';

const theme = {
  colors: {
    primary: '#3b82f6',
    secondary: '#6b7280',
    danger: '#ef4444',
    bg: '#ffffff',
    text: '#111827',
  },
  spacing: (n: number) => `${n * 4}px`,
  radius: '6px',
} as const;

type Theme = typeof theme;

declare module 'styled-components' {
  export interface DefaultTheme extends Theme {}
}

const Card = styled.div`
  background: ${({ theme }) => theme.colors.bg};
  color: ${({ theme }) => theme.colors.text};
  padding: ${({ theme }) => theme.spacing(4)};
  border-radius: ${({ theme }) => theme.radius};
`;

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Card>Themed card</Card>
    </ThemeProvider>
  );
}
```

## Emotion

### @emotion/styled (styled-components API)

```typescript
import styled from '@emotion/styled';

const Button = styled.button<{ primary?: boolean }>`
  background: ${({ primary }) => primary ? '#3b82f6' : '#e5e7eb'};
  color: ${({ primary }) => primary ? 'white' : '#111827'};
  padding: 8px 16px;
  border: none;
  border-radius: 6px;
`;
```

### css Prop

```typescript
/** @jsxImportSource @emotion/react */
import { css } from '@emotion/react';

const buttonStyle = css`
  background: #3b82f6;
  color: white;
  padding: 8px 16px;
`;

function App() {
  return <button css={buttonStyle}>Click</button>;
}
```

## Performance Considerations

### Runtime CSS-in-JS (styled-components, Emotion)

- Styles computed at runtime
- Adds to JavaScript bundle size
- Can cause hydration mismatches in SSR
- Extra work on every render for dynamic styles

### Zero-Runtime Alternatives

| Library | Approach | Bundle Impact |
| :-- | :-- | :-- |
| **Vanilla Extract** | Build-time CSS generation | Zero runtime |
| **Linaria** | Zero-runtime CSS-in-JS | Zero runtime |
| **Panda CSS** | Build-time with token system | Near-zero |
| **StyleX** (Meta) | Compile-time atomic CSS | Near-zero |

```typescript
// Vanilla Extract example
// styles.css.ts
import { style } from '@vanilla-extract/css';

export const button = style({
  background: '#3b82f6',
  color: 'white',
  padding: '8px 16px',
  border: 'none',
  borderRadius: '6px',
});

// Component.tsx
import { button } from './styles.css';

function Button() {
  return <button className={button}>Click</button>;
}
```

## Comparison: CSS-in-JS vs CSS Modules vs Tailwind

| Feature | CSS-in-JS | CSS Modules | Tailwind |
| :-- | :-- | :-- | :-- |
| **Dynamic styles** | Native | className toggling | Conditional classes |
| **Bundle size** | Larger (runtime) | Small | Small (purged) |
| **Theming** | Built-in | CSS variables | Config file |
| **TypeScript** | Full support | Typed modules | Class strings |
| **SSR** | Needs setup | Works natively | Works natively |
| **Performance** | Runtime cost | No runtime | No runtime |
| **DX** | Co-located | Separate files | Inline classes |
| **Learning curve** | Medium | Low | Medium |

### When to Use What

- **CSS-in-JS** -- heavy theming, dynamic styles, design system libraries
- **CSS Modules** -- simple scoping, SSR-first apps, minimal overhead
- **Tailwind** -- rapid prototyping, utility-first teams, consistent design
- **Zero-runtime** -- performance-critical apps wanting CSS-in-JS DX
