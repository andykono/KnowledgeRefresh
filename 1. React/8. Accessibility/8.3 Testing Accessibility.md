# Testing Accessibility

[[8. Accessibility.md]]

---

Automated and manual accessibility testing catches issues before they reach users. Use axe-core, jest-axe, React Testing Library's a11y queries, and screen reader testing to validate your components.

## axe-core

axe-core finds many WCAG violations automatically.

### Setup

```bash
npm install -D @axe-core/react jest-axe
```

### In Development (React)

```tsx
// main.tsx or App.tsx (development only)
import React from 'react';
import ReactDOM from 'react-dom/client';
import axe from '@axe-core/react';

if (import.meta.env.DEV) {
  axe(React, ReactDOM, 1000);
}

import App from './App';
// ...
```

This logs a11y issues to the console during development.

## jest-axe

Run axe in unit tests:

```bash
npm install -D jest-axe @types/jest-axe
```

```tsx
// Button.test.tsx
import { render, screen } from '@testing-library/react';
import { axe, toHaveNoViolations } from 'jest-axe';
import { Button } from './Button';

expect.extend(toHaveNoViolations);

describe('Button', () => {
  it('should have no accessibility violations', async () => {
    const { container } = render(<Button>Click me</Button>);
    const results = await axe(container);
    expect(results).toHaveNoViolations();
  });

  it('should be accessible with aria-label', async () => {
    const { container } = render(
      <button aria-label="Close">
        <span aria-hidden>×</span>
      </button>
    );
    const results = await axe(container);
    expect(results).toHaveNoViolations();
  });
});
```

### Testing Forms

```tsx
// LoginForm.test.tsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { axe, toHaveNoViolations } from 'jest-axe';
import { LoginForm } from './LoginForm';

expect.extend(toHaveNoViolations);

describe('LoginForm', () => {
  it('has no a11y violations', async () => {
    const { container } = render(
      <LoginForm onSubmit={jest.fn()} />
    );
    const results = await axe(container);
    expect(results).toHaveNoViolations();
  });

  it('shows error with role="alert"', async () => {
    render(<LoginForm onSubmit={jest.fn()} error="Invalid credentials" />);
    expect(screen.getByRole('alert')).toHaveTextContent('Invalid credentials');
  });
});
```

## React Testing Library A11y Queries

RTL encourages accessible queries. Prefer these over `getByTestId`:

| Query | Use |
| :-- | :-- |
| `getByRole` | Buttons, links, headings, etc. |
| `getByLabelText` | Form fields by label |
| `getByPlaceholderText` | Fallback (avoid if possible) |
| `getByText` | Non-interactive text |
| `getByTitle` | Elements with title attribute |

### Role-Based Queries

```tsx
import { render, screen } from '@testing-library/react';

// Prefer role queries - they mirror how assistive tech finds elements
const button = screen.getByRole('button', { name: /submit/i });
const link = screen.getByRole('link', { name: /home/i });
const heading = screen.getByRole('heading', { level: 1 });
const textbox = screen.getByRole('textbox', { name: /email/i });
const checkbox = screen.getByRole('checkbox', { name: /subscribe/i });
```

### Label Text

```tsx
// Matches <label htmlFor="email">Email</label> + <input id="email" />
const input = screen.getByLabelText(/email/i);
```

### Testing Keyboard Interaction

```tsx
import userEvent from '@testing-library/user-event';

it('closes modal on Escape', async () => {
  const onClose = jest.fn();
  render(<Modal isOpen onClose={onClose} />);
  await userEvent.keyboard('{Escape}');
  expect(onClose).toHaveBeenCalled();
});

it('navigates with Tab', async () => {
  render(<Form />);
  const first = screen.getByLabelText('First field');
  const second = screen.getByLabelText('Second field');
  first.focus();
  await userEvent.tab();
  expect(second).toHaveFocus();
});
```

## Screen Reader Testing

### VoiceOver (macOS)

1. Enable: Cmd + F5
2. Navigate: VO + Arrow keys
3. Interact: VO + Shift + Down
4. Stop: Cmd + F5

### NVDA (Windows)

1. Download from nvaccess.org
2. Navigate with arrow keys
3. Use Insert key for commands

### Manual Checklist

- [ ] All interactive elements reachable by Tab
- [ ] Focus order is logical
- [ ] Form labels announced correctly
- [ ] Error messages announced (role="alert")
- [ ] Modal traps focus and restores on close
- [ ] Images have alt text or aria-hidden

## WCAG Checklist (Level A/AA)

### Perceivable

- [ ] Text alternatives for non-text content
- [ ] Captions for audio/video
- [ ] Content can be presented in different ways
- [ ] Sufficient color contrast (4.5:1 for text)

### Operable

- [ ] All functionality available via keyboard
- [ ] No keyboard traps
- [ ] Skip links for repeated content
- [ ] Focus visible

### Understandable

- [ ] Page language set (`lang` attribute)
- [ ] Labels and instructions for forms
- [ ] Error identification and suggestions

### Robust

- [ ] Valid HTML
- [ ] ARIA used correctly
- [ ] Compatible with assistive technologies

## Common Violations and Fixes

| Violation | Fix |
| :-- | :-- |
| Missing form labels | Add `<label htmlFor="id">` or `aria-label` |
| Low contrast | Increase contrast to 4.5:1 (AA) |
| Missing alt text | Add `alt` to images or `aria-hidden="true"` for decorative |
| Focus not visible | Add `:focus-visible` styles |
| Empty links/buttons | Add `aria-label` or visible text |
| Invalid ARIA | Use correct roles and attributes |
