# ARIA and Semantic HTML

[[8. Accessibility.md]]

---

Semantic HTML and ARIA attributes make React applications usable by everyone, including people using screen readers, keyboard navigation, and other assistive technologies.

## Semantic HTML Elements

Use native HTML elements before reaching for ARIA -- they come with built-in accessibility.

```typescript
function PageLayout() {
  return (
    <>
      <header>
        <nav aria-label="Main navigation">
          <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
          </ul>
        </nav>
      </header>

      <main>
        <article>
          <h1>Article Title</h1>
          <section aria-labelledby="intro-heading">
            <h2 id="intro-heading">Introduction</h2>
            <p>Content here...</p>
          </section>
        </article>

        <aside aria-label="Related links">
          <h2>Related</h2>
          <ul>
            <li><a href="/related">Related article</a></li>
          </ul>
        </aside>
      </main>

      <footer>
        <p>&copy; 2026</p>
      </footer>
    </>
  );
}
```

### Element Reference

| Element | Purpose | Implicit ARIA Role |
| :-- | :-- | :-- |
| `<header>` | Page/section header | `banner` |
| `<nav>` | Navigation links | `navigation` |
| `<main>` | Main content | `main` |
| `<section>` | Thematic grouping | `region` (with label) |
| `<article>` | Self-contained content | `article` |
| `<aside>` | Tangential content | `complementary` |
| `<footer>` | Page/section footer | `contentinfo` |
| `<button>` | Clickable action | `button` |
| `<a href>` | Navigation link | `link` |

## ARIA Roles

Use `role` when semantic HTML isn't sufficient.

```typescript
// Custom button that needs role
function IconButton({ onClick, label }: { onClick: () => void; label: string }) {
  return (
    <div
      role="button"
      tabIndex={0}
      onClick={onClick}
      onKeyDown={(e) => {
        if (e.key === 'Enter' || e.key === ' ') {
          e.preventDefault();
          onClick();
        }
      }}
      aria-label={label}
    >
      <Icon />
    </div>
  );
}

// Better: just use a <button>
function IconButton({ onClick, label }: { onClick: () => void; label: string }) {
  return (
    <button onClick={onClick} aria-label={label}>
      <Icon />
    </button>
  );
}
```

### Common ARIA Roles

| Role | Purpose | Example |
| :-- | :-- | :-- |
| `alert` | Important message | Toast notifications |
| `alertdialog` | Alert requiring response | Confirmation modal |
| `dialog` | Dialog window | Modal |
| `tab` / `tablist` / `tabpanel` | Tab interface | Tab navigation |
| `menu` / `menuitem` | Dropdown menu | Context menu |
| `status` | Status update | Loading indicator |
| `search` | Search section | Search form |

## ARIA Attributes

### Labeling

```typescript
// aria-label: invisible label for screen readers
<button aria-label="Close dialog">
  <XIcon />
</button>

// aria-labelledby: reference a visible label
<h2 id="dialog-title">Settings</h2>
<div role="dialog" aria-labelledby="dialog-title">
  ...
</div>

// aria-describedby: additional description
<input
  id="email"
  aria-describedby="email-help email-error"
/>
<p id="email-help">We'll never share your email.</p>
<p id="email-error" role="alert">Invalid email format.</p>
```

### State Attributes

```typescript
// aria-expanded: collapsible sections
function Accordion({ title, children }: { title: string; children: React.ReactNode }) {
  const [isOpen, setIsOpen] = useState(false);
  const contentId = useId();

  return (
    <div>
      <button
        aria-expanded={isOpen}
        aria-controls={contentId}
        onClick={() => setIsOpen(!isOpen)}
      >
        {title}
      </button>
      <div id={contentId} role="region" hidden={!isOpen}>
        {children}
      </div>
    </div>
  );
}

// aria-selected: selected item in a list
<li role="option" aria-selected={isSelected}>
  {item.name}
</li>

// aria-disabled: visually present but not interactive
<button aria-disabled={isLoading} onClick={isLoading ? undefined : handleClick}>
  {isLoading ? 'Saving...' : 'Save'}
</button>
```

### Live Regions

Announce dynamic content changes to screen readers.

```typescript
// aria-live="polite" — announces when user is idle
<div aria-live="polite" aria-atomic="true">
  {statusMessage && <p>{statusMessage}</p>}
</div>

// aria-live="assertive" — announces immediately (interrupts)
<div role="alert" aria-live="assertive">
  {error && <p>{error}</p>}
</div>
```

## Accessible Form Patterns

```typescript
function LoginForm() {
  const [errors, setErrors] = useState<Record<string, string>>({});

  return (
    <form aria-label="Login form" onSubmit={handleSubmit}>
      <div>
        <label htmlFor="email">
          Email <span aria-hidden="true">*</span>
        </label>
        <input
          id="email"
          type="email"
          required
          aria-required="true"
          aria-invalid={!!errors.email}
          aria-describedby={errors.email ? 'email-error' : undefined}
        />
        {errors.email && (
          <p id="email-error" role="alert">
            {errors.email}
          </p>
        )}
      </div>

      <div>
        <label htmlFor="password">
          Password <span aria-hidden="true">*</span>
        </label>
        <input
          id="password"
          type="password"
          required
          aria-required="true"
          aria-invalid={!!errors.password}
          aria-describedby="password-help"
        />
        <p id="password-help">Must be at least 8 characters.</p>
      </div>

      <button type="submit">Log In</button>
    </form>
  );
}
```

## Accessible Tabs

```typescript
function Tabs({ tabs }: { tabs: { id: string; label: string; content: React.ReactNode }[] }) {
  const [activeTab, setActiveTab] = useState(tabs[0].id);

  return (
    <div>
      <div role="tablist" aria-label="Content tabs">
        {tabs.map((tab) => (
          <button
            key={tab.id}
            role="tab"
            id={`tab-${tab.id}`}
            aria-selected={activeTab === tab.id}
            aria-controls={`panel-${tab.id}`}
            tabIndex={activeTab === tab.id ? 0 : -1}
            onClick={() => setActiveTab(tab.id)}
          >
            {tab.label}
          </button>
        ))}
      </div>

      {tabs.map((tab) => (
        <div
          key={tab.id}
          role="tabpanel"
          id={`panel-${tab.id}`}
          aria-labelledby={`tab-${tab.id}`}
          hidden={activeTab !== tab.id}
          tabIndex={0}
        >
          {tab.content}
        </div>
      ))}
    </div>
  );
}
```

## Image Accessibility

```typescript
// Informative image — descriptive alt
<img src="/chart.png" alt="Sales increased 25% in Q4 2025" />

// Decorative image — empty alt
<img src="/divider.svg" alt="" />

// Complex image — longer description
<figure>
  <img src="/diagram.png" alt="System architecture overview" aria-describedby="diagram-desc" />
  <figcaption id="diagram-desc">
    The system uses a microservices architecture with an API gateway...
  </figcaption>
</figure>

// Icon with text — hide icon from screen readers
<button>
  <SearchIcon aria-hidden="true" />
  Search
</button>

// Icon without text — provide label
<button aria-label="Search">
  <SearchIcon aria-hidden="true" />
</button>
```

## Common Mistakes

| Mistake | Fix |
| :-- | :-- |
| `<div onClick>` instead of `<button>` | Use `<button>` for clickable elements |
| Missing `alt` on `<img>` | Always provide `alt` (empty for decorative) |
| `<a>` without `href` | Use `<button>` for actions, `<a href>` for navigation |
| Color-only indication | Add icons, text, or patterns alongside color |
| Missing form labels | Every input needs a `<label>` or `aria-label` |
| Auto-playing media | Provide controls and respect `prefers-reduced-motion` |
