# Accessibility

[[../README.md]]

---

Accessibility (a11y) ensures your React applications are usable by everyone, including people who use assistive technologies like screen readers, keyboard navigation, or voice control. Building accessible apps from the start is easier than retrofitting later.

## Overview

Web accessibility follows the **WCAG** (Web Content Accessibility Guidelines). React's component model and declarative UI make it well-suited for a11y when you use semantic HTML, ARIA correctly, and manage focus properly.

## Why Accessibility Matters

- **Legal** - ADA, Section 508, and similar laws require accessible digital experiences
- **Inclusive** - ~15% of people have some form of disability
- **SEO** - Semantic HTML and structure help search engines
- **Usability** - Keyboard users, low vision, and situational limitations benefit

## Quick Checklist

- [ ] Use semantic HTML (`<button>`, `<nav>`, `<main>`, `<article>`)
- [ ] Ensure keyboard navigation works (Tab, Enter, Escape)
- [ ] Add `aria-label` or visible labels for icons and controls
- [ ] Trap focus in modals; restore on close
- [ ] Run axe-core or jest-axe in tests
- [ ] Test with a screen reader (VoiceOver, NVDA)
