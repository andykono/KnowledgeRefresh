# Performance

[[../README.md]]

---

Comprehensive guide to React performance optimization. Covers rendering model, memoization, code splitting, virtualization, and Web Vitals.

## Overview

React performance optimization follows a systematic approach:

| Phase | Focus | Key Techniques |
| :-- | :-- | :-- |
| **Understanding** | How React renders | Trigger → Render → Commit, reconciliation |
| **Preventing** | Unnecessary work | memo, useMemo, useCallback |
| **Splitting** | Reduce initial load | React.lazy, Suspense, dynamic imports |
| **Rendering** | Large lists/tables | Virtualization, windowing |
| **Measuring** | Real user experience | Web Vitals, Profiler, Lighthouse |

---

## Performance Checklist

- [ ] Understand what causes re-renders
- [ ] Use React.memo for expensive leaf components
- [ ] Memoize expensive computations with useMemo
- [ ] Stabilize callbacks with useCallback (or ref pattern)
- [ ] Implement route-based code splitting
- [ ] Add Suspense boundaries for lazy components
- [ ] Virtualize lists with 100+ items
- [ ] Measure LCP, INP, CLS with web-vitals
- [ ] Profile with React DevTools Profiler
- [ ] Run Lighthouse audits regularly

---

## Optimization Strategy

```
1. Measure first (don't guess)
   └─ React Profiler, Chrome DevTools, web-vitals

2. Identify bottlenecks
   ├─ Too many re-renders? → Memoization
   ├─ Large initial bundle? → Code splitting
   ├─ Long lists? → Virtualization
   └─ Slow LCP/INP/CLS? → Web Vitals fixes

3. Apply targeted fixes
   └─ One change at a time, measure again

4. Consider React Compiler (future)
   └─ Automatic memoization when stable
```

## Key Principles

1. **Measure before optimizing** — Use Profiler and Web Vitals to find real bottlenecks
2. **Memoization has a cost** — Only use when you've identified unnecessary re-renders
3. **Code splitting reduces TTI** — Split by route and heavy components
4. **Virtualization for scale** — Essential for 100+ list items
5. **Web Vitals = UX** — LCP, INP, CLS directly impact user perception
