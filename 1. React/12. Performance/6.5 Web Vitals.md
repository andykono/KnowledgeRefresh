# Web Vitals

[[6. Performance.md]]

---

Web Vitals are user-centric metrics that measure real-world performance. Google prioritizes LCP, INP, and CLS.

## Core Web Vitals

| Metric | Full Name | Measures | Good | Needs Improvement | Poor |
| :-- | :-- | :-- | :-- | :-- | :-- |
| **LCP** | Largest Contentful Paint | Loading | ≤2.5s | 2.5–4s | >4s |
| **INP** | Interaction to Next Paint | Responsiveness | ≤200ms | 200–500ms | >500ms |
| **CLS** | Cumulative Layout Shift | Visual stability | ≤0.1 | 0.1–0.25 | >0.25 |

---

## LCP (Largest Contentful Paint)

Measures when the largest visible content element (image, video, text block) is rendered.

### What Affects LCP

- Server response time (TTFB)
- Render-blocking resources
- Client-side rendering delay
- Large images/media

### Measuring with web-vitals

```bash
npm install web-vitals
```

```typescript
// reportWebVitals.ts
import { onLCP, onINP, onCLS } from 'web-vitals';

function sendToAnalytics(metric: { name: string; value: number; id: string }) {
  // Send to your analytics
  console.log(metric);
  // analytics.track('web_vital', metric);
}

export function reportWebVitals() {
  onLCP(sendToAnalytics);
  onINP(sendToAnalytics);
  onCLS(sendToAnalytics);
}
```

```typescript
// main.tsx
import { reportWebVitals } from './reportWebVitals';

const root = createRoot(document.getElementById('root')!);
root.render(<App />);
reportWebVitals();
```

### LCP Optimization Strategies

```typescript
// 1. Preload critical resources
<link rel="preload" href="/hero-image.jpg" as="image" />

// 2. Optimize images
<img
  src="/hero.webp"
  width={1200}
  height={600}
  loading="eager"
  fetchPriority="high"
  alt="Hero"
/>

// 3. Reduce JavaScript for above-the-fold content
// Use code splitting so LCP element doesn't wait for full bundle

// 4. Use SSR or static generation for LCP content
// Server-rendered hero loads faster than client-rendered
```

## INP (Interaction to Next Paint)

Replaces FID (First Input Delay). Measures responsiveness — time from user interaction to next paint.

### What Affects INP

- Long JavaScript tasks blocking main thread
- Heavy component re-renders
- Large DOM updates

### INP Optimization Strategies

```typescript
// 1. Debounce/throttle expensive handlers
const handleSearch = useMemo(
  () => debounce((query: string) => search(query), 300),
  []
);

// 2. Use useTransition for non-urgent updates
const [isPending, startTransition] = useTransition();
const handleFilter = (value: string) => {
  startTransition(() => {
    setFilter(value);
  });
};

// 3. Break up long tasks with requestIdleCallback
function processInChunks(items: Item[], process: (item: Item) => void) {
  let index = 0;
  function work(deadline: IdleDeadline) {
    while (index < items.length && deadline.timeRemaining() > 0) {
      process(items[index++]);
    }
    if (index < items.length) {
      requestIdleCallback(work);
    }
  }
  requestIdleCallback(work);
}

// 4. Virtualize long lists (see [[6.4 Virtualization.md]])
```

## CLS (Cumulative Layout Shift)

Measures unexpected layout shifts. Elements moving as content loads hurts UX.

### What Causes CLS

- Images without dimensions
- Ads/embeds injected dynamically
- Web fonts causing FOIT/FOUT
- Dynamically injected content above existing content

### CLS Optimization Strategies

```typescript
// 1. Always set dimensions on images and video
<img
  src="/photo.jpg"
  width={800}
  height={600}
  alt="Description"
  style={{ aspectRatio: '800/600' }}
/>

// 2. Reserve space for dynamic content
const [content, setContent] = useState<string | null>(null);
return (
  <div style={{ minHeight: 200 }}>
    {content ?? <Skeleton />}
  </div>
);

// 3. Use transform for animations (doesn't trigger layout)
// BAD: top, left, width, height
// GOOD: transform: translateX()

// 4. Preload fonts to avoid FOUT
<link
  rel="preload"
  href="/fonts/main.woff2"
  as="font"
  type="font/woff2"
  crossOrigin="anonymous"
/>
```

## React Profiler API

Measure component render performance in development.

```typescript
import { Profiler } from 'react';

function onRenderCallback(
  id: string,
  phase: 'mount' | 'update',
  actualDuration: number,
  baseDuration: number,
  startTime: number,
  commitTime: number
) {
  console.log(`${id} (${phase}) took ${actualDuration}ms`);
}

function App() {
  return (
    <Profiler id="App" onRender={onRenderCallback}>
      <Dashboard />
    </Profiler>
  );
}
```

## Chrome DevTools Performance Tab

1. Open DevTools → Performance
2. Click Record
3. Interact with the page
4. Stop recording
5. Analyze: Long tasks, layout thrashing, JavaScript execution

### Key Things to Look For

- **Long tasks** (>50ms) — Break up or defer
- **Layout thrashing** — Read/write/read/write pattern
- **Forced reflows** — Avoid reading layout (offsetHeight, getBoundingClientRect) after writes

## Lighthouse

Run in Chrome DevTools (Lighthouse tab) or CLI:

```bash
npx lighthouse https://yoursite.com --view
```

### Key Audits

- Performance score (includes LCP, INP, CLS)
- "Avoid enormous network payloads"
- "Reduce unused JavaScript"
- "Properly size images"
