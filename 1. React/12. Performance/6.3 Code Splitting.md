# Code Splitting

[[6. Performance.md]]

---

Code splitting reduces initial bundle size by loading code only when needed. React provides `lazy` and `Suspense` for this.

## React.lazy

`React.lazy` enables dynamic imports for components. The component is loaded when first rendered.

```typescript
import { lazy } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));
const AdminPanel = lazy(() => import('./pages/AdminPanel'));
```

### Requirements

- Lazy components must default-export a React component
- Must be wrapped in `Suspense` (or Error Boundary handles loading state)

```typescript
// Dashboard.tsx — must have default export
export default function Dashboard() {
  return <div>Dashboard</div>;
}
```

---

## Suspense Boundaries

`Suspense` shows a fallback while lazy children load.

```typescript
import { Suspense, lazy } from 'react';

const HeavyChart = lazy(() => import('./HeavyChart'));

function AnalyticsPage() {
  return (
    <div>
      <h1>Analytics</h1>
      <Suspense fallback={<ChartSkeleton />}>
        <HeavyChart />
      </Suspense>
    </div>
  );
}
```

### Multiple Lazy Components

```typescript
function App() {
  return (
    <Suspense fallback={<PageLoader />}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/dashboard" element={
          <Suspense fallback={<DashboardSkeleton />}>
            <Dashboard />
          </Suspense>
        } />
      </Routes>
    </Suspense>
  );
}
```

## Route-Based Splitting

Split by route so each page loads its own chunk.

```typescript
// routes.tsx
import { lazy, Suspense } from 'react';
import { Routes, Route } from 'react-router-dom';

const Home = lazy(() => import('./pages/Home'));
const Products = lazy(() => import('./pages/Products'));
const ProductDetail = lazy(() => import('./pages/ProductDetail'));
const Cart = lazy(() => import('./pages/Cart'));
const Checkout = lazy(() => import('./pages/Checkout'));

function AppRoutes() {
  return (
    <Suspense fallback={<FullPageSpinner />}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/products" element={<Products />} />
        <Route path="/products/:id" element={<ProductDetail />} />
        <Route path="/cart" element={<Cart />} />
        <Route path="/checkout" element={<Checkout />} />
      </Routes>
    </Suspense>
  );
}
```

## Component-Based Splitting

Split heavy components used conditionally (modals, tabs, charts).

```typescript
const RichTextEditor = lazy(() => import('./RichTextEditor'));
const PDFViewer = lazy(() => import('./PDFViewer'));

function DocumentPage({ type }: { type: 'edit' | 'view' }) {
  return (
    <div>
      {type === 'edit' && (
        <Suspense fallback={<EditorSkeleton />}>
          <RichTextEditor />
        </Suspense>
      )}
      {type === 'view' && (
        <Suspense fallback={<ViewerSkeleton />}>
          <PDFViewer />
        </Suspense>
      )}
    </div>
  );
}
```

## Dynamic Imports

Use dynamic `import()` for non-React code (libraries, utilities).

```typescript
async function handleExport() {
  const { exportToPDF } = await import('./utils/pdfExport');
  await exportToPDF(data);
}

// Lazy load a heavy library
async function analyzeData(data: Data[]) {
  const { analyze } = await import('heavy-analytics-lib');
  return analyze(data);
}
```

## Preloading

Load chunks before the user navigates to improve perceived performance.

```typescript
// Preload on hover or when route is likely
const Dashboard = lazy(() => import('./pages/Dashboard'));

function NavLink({ to, children }: { to: string; children: React.ReactNode }) {
  const preload = () => {
    if (to === '/dashboard') {
      import('./pages/Dashboard');
    }
  };

  return (
    <Link to={to} onMouseEnter={preload} onFocus={preload}>
      {children}
    </Link>
  );
}
```

### Preload with Route Config

```typescript
const routeConfig = [
  { path: '/dashboard', component: () => import('./pages/Dashboard') },
  { path: '/settings', component: () => import('./pages/Settings') },
];

// Preload all route chunks on app load (aggressive)
routeConfig.forEach(({ component }) => component());
```

## Best Practices

1. **Split by route** — Each page = separate chunk
2. **Name chunks** — Use magic comments for predictable names

```typescript
const Dashboard = lazy(() =>
  import(/* webpackChunkName: "dashboard" */ './pages/Dashboard')
);
```

3. **Meaningful fallbacks** — Match fallback to component shape (skeleton > spinner)
4. **Avoid over-splitting** — Too many tiny chunks increase HTTP overhead
5. **Preload critical paths** — Hover on nav links for likely next page
