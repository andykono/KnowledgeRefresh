React apps are made of **components** — JavaScript functions that return JSX.

## Functional Components (Recommended)

```typescript
// Simple functional component
function Greeting() {
  return <h1>Hello, World!</h1>;
}

// With props
interface GreetingProps {
  name: string;
  age?: number;  // Optional
}

function Greeting({ name, age }: GreetingProps) {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      {age && <p>Age: {age}</p>}
    </div>
  );
}

// Export for use
export default Greeting;
```

## Class Components (Legacy)

```typescript
// Class-based component (older pattern)
class Greeting extends React.Component<GreetingProps> {
  render() {
    const { name, age } = this.props;
    return (
      <div>
        <h1>Hello, {name}!</h1>
        {age && <p>Age: {age}</p>}
      </div>
    );
  }
}

export default Greeting;
```

**⚠️ Functional components are recommended.** Class components are legacy and mostly unused in modern React.

## Component Patterns

### Stateless Presentational Component
```typescript
// Just displays props, no state or logic
function Badge({ label, variant = 'primary' }) {
  return <span className={`badge badge-${variant}`}>{label}</span>;
}
```

### Container + Presentational
```typescript
// Container (logic)
function UserContainer() {
  const [user, setUser] = useState(null);
  useEffect(() => {
    fetchUser().then(setUser);
  }, []);
  return <UserDisplay user={user} />;
}

// Presentational (display)
function UserDisplay({ user }) {
  if (!user) return <div>Loading...</div>;
  return <div>{user.name}</div>;
}
```

### Wrapper Component
```typescript
interface CardProps {
  children: React.ReactNode;
  title: string;
}

function Card({ children, title }: CardProps) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div className="card-content">{children}</div>
    </div>
  );
}

// Usage
<Card title="User Info">
  <p>Name: John Doe</p>
  <p>Email: john@example.com</p>
</Card>
```

## Component Composition

### Props Children Pattern
```typescript
function Layout({ children }) {
  return (
    <div className="layout">
      <header>Header</header>
      <main>{children}</main>
      <footer>Footer</footer>
    </div>
  );
}

// Usage
<Layout>
  <h1>Welcome</h1>
  <p>Main content here</p>
</Layout>
```

### Slots Pattern
```typescript
interface ModalProps {
  header?: React.ReactNode;
  body: React.ReactNode;
  footer?: React.ReactNode;
}

function Modal({ header, body, footer }: ModalProps) {
  return (
    <div className="modal">
      {header && <div className="modal-header">{header}</div>}
      <div className="modal-body">{body}</div>
      {footer && <div className="modal-footer">{footer}</div>}
    </div>
  );
}
```

## Component Best Practices

✅ **DO:**
- Keep components small and focused
- Use TypeScript for props
- Name components with PascalCase
- Extract reusable components
- Use functional components with hooks

❌ **DON'T:**
- Don't put all logic in one component
- Don't name components with lowercase
- Don't mutate props directly
- Don't pass all state down as props (prop drilling)

## Component Lifecycle

### Class Component Lifecycle (Legacy)
```typescript
class Component extends React.Component {
  componentDidMount() {
    // After render
  }

  componentDidUpdate(prevProps, prevState) {
    // After update
  }

  componentWillUnmount() {
    // Before remove
  }
}
```

### Functional Component with Hooks (Modern)
```typescript
function Component() {
  // Mount + Update
  useEffect(() => {
    // Setup
    return () => {
      // Cleanup (like componentWillUnmount)
    };
  }, [dependencies]);
}
```

See: [[2. React Hooks/2.3 useEffect.md|useEffect hook]] for details.

## Quick Comparison

| Aspect | Functional | Class |
| :-- | :-- | :-- |
| **Syntax** | Function | extends Component |
| **State** | useState hook | this.state |
| **Lifecycle** | useEffect hook | componentDidMount, etc. |
| **this binding** | ✅ Not needed | ❌ Required |
| **Performance** | ⭐ Better | ⭐ Older |
| **Recommended** | ✅ YES | ❌ Legacy |
