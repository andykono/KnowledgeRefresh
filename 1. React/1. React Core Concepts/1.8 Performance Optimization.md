# Performance Optimization

[[1. React Core Concepts.md]]

---

Techniques to make React apps faster.

## Why Components Re-render

A component re-renders when:

1. **Its own state changes** — `useState`, `useReducer`
2. **Its parent re-renders** — children re-render by default (unless memoized)
3. **Context it consumes changes** — any `useContext` value update triggers re-renders for all consumers
4. **Props change** — new props cause a re-render

```typescript
// Parent re-renders → Child re-renders (even if props unchanged)
function Parent() {
  const [count, setCount] = useState(0);
  return (
    <>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
      <Child name="Alice" />  {/* Re-renders when count changes */}
    </>
  );
}
```

## Memoization

```typescript
// Prevent re-renders with memo
const MemoComponent = memo(({ data }) => {
  return <div>{data.name}</div>;
});

// Memoize expensive values
const expensiveValue = useMemo(() => {
  return calculations(input);
}, [input]);

// Memoize callbacks
const handleClick = useCallback(() => {
  // ...
}, [dependencies]);
```

## React.memo Deep Dive

`memo` does a shallow prop comparison. If props are referentially equal, React skips the re-render.

### When It Helps

- Expensive child components that receive stable props
- Lists where most items don't change
- Components that re-render often due to parent state

### When It Doesn't Help

- Props change every render (e.g., inline objects, new function references)
- Component is cheap to render
- Parent always passes new references

### Custom Comparator

```typescript
const UserCard = memo(({ user }: { user: User }) => (
  <div>{user.name}</div>
), (prevProps, nextProps) => {
  return prevProps.user.id === nextProps.user.id;
});
```

Return `true` if props are equal (skip re-render); `false` if they differ (re-render).

## useMemo and useCallback

### Correct Usage

```typescript
// useMemo: expensive computation
const sortedItems = useMemo(
  () => items.slice().sort((a, b) => a.name.localeCompare(b.name)),
  [items]
);

// useCallback: pass stable callback to memoized child
const handleSubmit = useCallback((data: FormData) => {
  submitData(data);
}, [submitData]);
```

### Over-use Anti-pattern

```typescript
// ❌ Unnecessary: cheap computation
const doubled = useMemo(() => count * 2, [count]);

// ❌ Unnecessary: callback not passed to memoized child
const handleClick = useCallback(() => setCount(c => c + 1), []);

// ✅ useMemo/useCallback add overhead; use only when needed
```

Use them when: passing to memoized children, as effect dependencies, or for expensive calculations.

## Code Splitting with lazy/Suspense

```typescript
// Lazy load components
const HeavyComponent = lazy(() => import('./Heavy'));

// Usage with Suspense
<Suspense fallback={<div>Loading...</div>}>
  <HeavyComponent />
</Suspense>
```

Route-based splitting:

```typescript
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));

<Routes>
  <Route path="/" element={
    <Suspense fallback={<PageLoader />}>
      <Dashboard />
    </Suspense>
  } />
  <Route path="/settings" element={
    <Suspense fallback={<PageLoader />}>
      <Settings />
    </Suspense>
  } />
</Routes>
```

## List Virtualization

For long lists (100+ items), render only visible rows:

```typescript
import { useVirtualizer } from '@tanstack/react-virtual';

function VirtualList({ items }: { items: string[] }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
  });

  return (
    <div ref={parentRef} style={{ height: 400, overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize() }}>
        {virtualizer.getVirtualItems().map((virtualRow) => (
          <div
            key={virtualRow.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              transform: `translateY(${virtualRow.start}px)`,
            }}
          >
            {items[virtualRow.index]}
          </div>
        ))}
      </div>
    </div>
  );
}
```

Libraries: `@tanstack/react-virtual`, `react-window`, `react-virtuoso`.

## Debouncing and Throttling in React

```typescript
// Debounce search input
function SearchInput() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDeferredValue(query);  // React 18+

  useEffect(() => {
    if (!debouncedQuery) return;
    fetchResults(debouncedQuery);
  }, [debouncedQuery]);

  return <input value={query} onChange={(e) => setQuery(e.target.value)} />;
}
```

Custom debounce hook:

```typescript
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
}
```

## React DevTools Profiler Usage

1. Open React DevTools → **Profiler** tab
2. Click **Record**, interact with the app, then **Stop**
3. Inspect:
   - **Flame graph** — which components rendered and how long
   - **Ranked** — components sorted by render duration
   - **Why did this render?** — reasons (props, state, parent)

Use this to find unnecessary re-renders and expensive components.

## Common Performance Mistakes

1. **Creating objects/arrays inline** — new reference every render, breaks memo
   ```typescript
   // ❌
   <Child style={{ color: 'red' }} items={[1, 2, 3]} />
   // ✅
   const style = useMemo(() => ({ color: 'red' }), []);
   const items = useMemo(() => [1, 2, 3], []);
   ```

2. **Inline functions as props** — new function every render
   ```typescript
   // ❌
   <Child onClick={() => doSomething()} />
   // ✅
   const handleClick = useCallback(() => doSomething(), []);
   ```

3. **Over-splitting context** — one big context forces many re-renders; split by update frequency
4. **Missing keys or bad keys** — see [[1.5 Keys and Refs.md]]
5. **Not measuring first** — optimize only after profiling; premature optimization adds complexity

---

## Key Performance Tips

1. **Remove unnecessary re-renders** — Use memo
2. **Memoize expensive calculations** — Use useMemo
3. **Split large components** — Smaller is faster
4. **Use key prop correctly** — Helps React identify items
5. **Defer updates** — useTransition, useDeferredValue
6. **Lazy load** — Code splitting with lazy()
7. **Virtual lists** — For very long lists (100+)
