# Lazy Routes

[[3. Routing.md]]

---

Route-based code splitting reduces initial bundle size by loading route components only when their routes are visited. Use `React.lazy()` and `Suspense` for this.

## Basic Lazy Loading

```typescript
import { lazy, Suspense } from 'react';
import { Routes, Route } from 'react-router-dom';

const HomePage = lazy(() => import('./pages/HomePage'));
const AboutPage = lazy(() => import('./pages/AboutPage'));
const DashboardPage = lazy(() => import('./pages/DashboardPage'));

function App() {
  return (
    <Suspense fallback={<PageLoader />}>
      <Routes>
        <Route path="/" element={<HomePage />} />
        <Route path="/about" element={<AboutPage />} />
        <Route path="/dashboard" element={<DashboardPage />} />
      </Routes>
    </Suspense>
  );
}
```

**How it works:** Each `lazy()` import becomes a separate chunk. The chunk loads when the route is first visited.

---

## Fallback Component

```typescript
function PageLoader() {
  return (
    <div className="page-loader">
      <Spinner />
      <p>Loading...</p>
    </div>
  );
}

<Suspense fallback={<PageLoader />}>
  <Routes>{/* ... */}</Routes>
</Suspense>
```

**Best practice:** Use a skeleton or spinner that matches your layout. Avoid empty or jarring transitions.

## Named Exports

`lazy()` expects default exports. For named exports:

```typescript
// ❌ Won't work: named export
export function DashboardPage() { ... }

// ✅ Option 1: Re-export as default in route file
// pages/DashboardPage.tsx
export { DashboardPage as default } from './DashboardPage';

// ✅ Option 2: Lazy with named import
const DashboardPage = lazy(() =>
  import('./pages/DashboardPage').then((m) => ({ default: m.DashboardPage }))
);
```

## Per-Route Suspense

Wrap each route in its own Suspense for finer control:

```typescript
<Routes>
  <Route
    path="/"
    element={
      <Suspense fallback={<HomeSkeleton />}>
        <HomePage />
      </Suspense>
    }
  />
  <Route
    path="/dashboard"
    element={
      <Suspense fallback={<DashboardSkeleton />}>
        <DashboardPage />
      </Suspense>
    }
  />
</Routes>
```

**When to use:** Different loading UIs per section, or when one heavy route shouldn't block others.

## Layout + Lazy Children

```typescript
const DashboardLayout = lazy(() => import('./layouts/DashboardLayout'));
const DashboardHome = lazy(() => import('./pages/DashboardHome'));
const SettingsPage = lazy(() => import('./pages/SettingsPage'));

<Routes>
  <Route
    path="/dashboard"
    element={
      <Suspense fallback={<LayoutSkeleton />}>
        <DashboardLayout />
      </Suspense>
    }
  >
    <Route
      index
      element={
        <Suspense fallback={<ContentSkeleton />}>
          <DashboardHome />
        </Suspense>
      }
    />
    <Route
      path="settings"
      element={
        <Suspense fallback={<ContentSkeleton />}>
          <SettingsPage />
        </Suspense>
      }
    />
  </Route>
</Routes>
```

## Error Boundary for Lazy

Lazy components can fail to load (network error, etc.). Wrap in an Error Boundary:

```typescript
import { Component, ErrorInfo, ReactNode } from 'react';

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

class RouteErrorBoundary extends Component<Props, { hasError: boolean }> {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  componentDidCatch(error: Error, info: ErrorInfo) {
    console.error('Route load failed:', error, info);
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback ?? <div>Failed to load page. <a href="/">Retry</a></div>;
    }
    return this.props.children;
  }
}

// Usage
<RouteErrorBoundary fallback={<LoadErrorPage />}>
  <Suspense fallback={<PageLoader />}>
    <Routes>{/* ... */}</Routes>
  </Suspense>
</RouteErrorBoundary>
```

## Preloading Routes

Load a route chunk before the user navigates:

```typescript
// Preload on hover
<Link
  to="/dashboard"
  onMouseEnter={() => import('./pages/DashboardPage')}
>
  Dashboard
</Link>

// Preload after initial load
useEffect(() => {
  const timer = setTimeout(() => {
    import('./pages/DashboardPage');
  }, 2000);
  return () => clearTimeout(timer);
}, []);
```

**When to use:** Likely next pages (e.g. dashboard after login).

## Lazy with Data Loader

If using React Router loaders, the loader runs before the component. Lazy still helps with component code:

```typescript
const UserProfile = lazy(() => import('./pages/UserProfile'));

const router = createBrowserRouter([
  {
    path: '/users/:userId',
    element: (
      <Suspense fallback={<ProfileSkeleton />}>
        <UserProfile />
      </Suspense>
    ),
    loader: async ({ params }) => {
      const user = await fetchUser(params.userId!);
      return { user };
    },
  },
]);
```

## Vite / Webpack Chunk Names

```typescript
// Vite
const DashboardPage = lazy(() =>
  import(/* @vite-ignore */ './pages/DashboardPage')
);

// Webpack
const DashboardPage = lazy(() =>
  import(/* webpackChunkName: "dashboard" */ './pages/DashboardPage')
);
```

## When to Use Lazy Routes

| Scenario | Use Lazy? |
| :-- | :-- |
| Small app (< 5 routes) | Optional |
| Medium/large app | ✅ Yes |
| Admin/dashboard (rarely visited) | ✅ Yes |
| Heavy charts, editors | ✅ Yes |
| Critical above-the-fold page | ❌ No (eager load) |

---

## Quick Reference

| API | Purpose |
| :-- | :-- |
| **lazy(() => import(...))** | Defer loading component |
| **Suspense** | Show fallback while loading |
| **fallback** | Loading UI during chunk fetch |
| **Error Boundary** | Handle load failures |
