# Nested and Dynamic Routes

[[3. Routing.md]]

---

Nested routes let you compose layouts (header, sidebar) and render child content via `Outlet`. Dynamic segments, optional params, and catch-all routes handle flexible URL patterns.

## Outlet

`Outlet` renders the matched child route. Use it in layout components:

```typescript
import { Outlet } from 'react-router-dom';

function MainLayout() {
  return (
    <div className="layout">
      <header>
        <nav>{/* ... */}</nav>
      </header>
      <main>
        <Outlet />  {/* Child route renders here */}
      </main>
      <footer>© 2025</footer>
    </div>
  );
}
```

**When to use:** Any layout that wraps multiple pages (dashboard, app shell, etc.).

---

## Nested Routes

Define parent and children in the same `Route`:

```typescript
<Routes>
  <Route path="/" element={<MainLayout />}>
    <Route index element={<HomePage />} />
    <Route path="about" element={<AboutPage />} />
    <Route path="users" element={<UsersLayout />}>
      <Route index element={<UserList />} />
      <Route path=":userId" element={<UserDetail />} />
    </Route>
  </Route>
</Routes>
```

**URL structure:**
- `/` → MainLayout + HomePage
- `/about` → MainLayout + AboutPage
- `/users` → MainLayout + UsersLayout + UserList
- `/users/123` → MainLayout + UsersLayout + UserDetail

**UsersLayout** must also render `<Outlet />` for its children.

---

## Dynamic Segments

Use `:paramName` for dynamic URL parts:

```typescript
<Route path="/users/:userId" element={<UserProfile />} />
<Route path="/posts/:postId/comments/:commentId" element={<CommentDetail />} />
```

```typescript
function UserProfile() {
  const { userId } = useParams<{ userId: string }>();
  return <div>User: {userId}</div>;
}
```

**When to use:** Detail pages, resource IDs, slugs.

---

## Optional Parameters

Wrap a segment in `?` for optional params (React Router v6.4+):

```typescript
// Matches /users and /users/123
<Route path="/users/:userId?" element={<UsersPage />} />
```

```typescript
function UsersPage() {
  const { userId } = useParams<{ userId?: string }>();

  if (userId) {
    return <UserDetail userId={userId} />;
  }
  return <UserList />;
}
```

**Alternative:** Use two routes instead of optional params for clearer separation.

---

## Catch-All Routes

Use `*` to match any remaining path:

```typescript
// Matches /docs, /docs/getting-started, /docs/api/react
<Route path="/docs/*" element={<DocsLayout />} />
```

```typescript
function DocsLayout() {
  const { '*': splat } = useParams<{ '*': string }>();
  // splat = "" | "getting-started" | "api/react"

  return (
    <div>
      <DocsSidebar />
      <Outlet />
    </div>
  );
}
```

**When to use:** Documentation sites, file-tree-style URLs, 404 fallbacks.

---

## Index Routes

`index` routes render at the parent path (no extra segment):

```typescript
<Route path="/dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHome />} />   {/* /dashboard */}
  <Route path="settings" element={<Settings />} />  {/* /dashboard/settings */}
  <Route path="reports" element={<Reports />} />   {/* /dashboard/reports */}
</Route>
```

**When to use:** Default child when parent path is visited (e.g. `/dashboard` shows overview).

## Layout Pattern

```typescript
// App.tsx
function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route element={<RootLayout />}>
          <Route path="/" element={<HomePage />} />
          <Route path="/products" element={<ProductsLayout />}>
            <Route index element={<ProductList />} />
            <Route path=":productId" element={<ProductDetail />} />
          </Route>
          <Route path="/about" element={<AboutPage />} />
        </Route>
        <Route path="/login" element={<LoginPage />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}

function RootLayout() {
  return (
    <div>
      <Header />
      <Outlet />
      <Footer />
    </div>
  );
}

function ProductsLayout() {
  return (
    <div className="products">
      <ProductsSidebar />
      <div className="content">
        <Outlet />
      </div>
    </div>
  );
}
```

## Relative vs Absolute Paths

In nested routes, paths are relative to the parent:

```typescript
<Route path="users" element={<UsersLayout />}>
  <Route path=":id" element={<UserDetail />} />  {/* users/:id */}
</Route>
```

Use leading `/` for absolute paths (relative to root):

```typescript
<Route path="/users" element={<UsersLayout />}>
  <Route path="/users/:id" element={<UserDetail />} />  {/* Explicit absolute */}
</Route>
```

**Best practice:** Prefer relative paths in nested routes for simpler refactoring.

## useMatch

Check if current route matches a path pattern:

```typescript
import { useMatch } from 'react-router-dom';

function NavItem({ to, children }: { to: string; children: React.ReactNode }) {
  const match = useMatch({ path: to, end: to === '/' });

  return (
    <Link to={to} className={match ? 'active' : ''}>
      {children}
    </Link>
  );
}
```

**When to use:** Custom active-state logic, conditional rendering based on route.

---

## useResolvedPath

Resolve a path relative to current location:

```typescript
import { useResolvedPath } from 'react-router-dom';

function BreadcrumbLink({ to, children }: { to: string; children: React.ReactNode }) {
  const resolved = useResolvedPath(to);
  return <Link to={resolved.pathname}>{children}</Link>;
}
```

**When to use:** Building links in nested route contexts.

## Data Router (loaders)

React Router v6.4+ supports loaders for data fetching before render:

```typescript
import { createBrowserRouter, RouterProvider } from 'react-router-dom';

const router = createBrowserRouter([
  {
    path: '/users/:userId',
    element: <UserProfile />,
    loader: async ({ params }) => {
      const user = await fetchUser(params.userId!);
      return { user };
    },
  },
]);

function App() {
  return <RouterProvider router={router} />;
}

function UserProfile() {
  const { user } = useLoaderData() as { user: User };
  return <div>{user.name}</div>;
}
```

**When to use:** Pre-load data before route renders, avoid loading spinners in child components.

## TypeScript: Route Params

```typescript
// Single param
const { id } = useParams<{ id: string }>();

// Optional param
const { tab } = useParams<{ tab?: string }>();

// Multiple params
const { userId, postId } = useParams<{ userId: string; postId: string }>();

// Catch-all
const { '*': splat } = useParams<{ '*': string }>();
```
