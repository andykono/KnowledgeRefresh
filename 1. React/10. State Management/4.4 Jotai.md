# Jotai

[[4. State Management.md]]

---

Jotai uses atoms — the smallest unit of state. State is bottom-up: compose atoms to build complex state. No providers required by default.

## Installation

```bash
npm install jotai
```

## Atoms

```typescript
import { atom, useAtom } from 'jotai';

const countAtom = atom(0);

function Counter() {
  const [count, setCount] = useAtom(countAtom);

  return (
    <div>
      <span>{count}</span>
      <button onClick={() => setCount((c) => c + 1)}>+</button>
    </div>
  );
}
```

## Derived Atoms (Read-only)

Compute state from other atoms. Only re-computes when dependencies change.

```typescript
import { atom, useAtomValue } from 'jotai';

const countAtom = atom(0);
const doubledAtom = atom((get) => get(countAtom) * 2);
const isEvenAtom = atom((get) => get(countAtom) % 2 === 0);

function Display() {
  const doubled = useAtomValue(doubledAtom);
  const isEven = useAtomValue(isEvenAtom);

  return (
    <div>
      Doubled: {doubled}, Even: {isEven ? 'yes' : 'no'}
    </div>
  );
}
```

## Writable Derived Atoms

Derived atom that can also write back to source atoms.

```typescript
const countAtom = atom(0);

const doubledAtom = atom(
  (get) => get(countAtom) * 2,
  (get, set, newValue: number) => {
    set(countAtom, newValue / 2);
  }
);

// Usage: setting doubledAtom updates countAtom
const [doubled, setDoubled] = useAtom(doubledAtom);
setDoubled(10); // countAtom becomes 5
```

## Async Atoms

Atoms can hold promises. Jotai integrates with Suspense.

```typescript
import { atom } from 'jotai';

const userIdAtom = atom(1);

const userAtom = atom(async (get) => {
  const userId = get(userIdAtom);
  const res = await fetch(`https://api.example.com/users/${userId}`);
  return res.json();
});

function UserProfile() {
  const user = useAtomValue(userAtom); // Suspends until resolved

  return <div>{user.name}</div>;
}

// With loading/error (no Suspense)
const userAtomWithStatus = atom((get) => {
  return get(userAtom);
});
```

## atomWithStorage

Persist atoms to localStorage or sessionStorage.

```typescript
import { atomWithStorage } from 'jotai/utils';

const themeAtom = atomWithStorage<'light' | 'dark'>('theme', 'light');

const settingsAtom = atomWithStorage('settings', {
  notifications: true,
  language: 'en',
});
```

## atomWithDefault

Atom with a default that can be reset.

```typescript
import { atomWithDefault } from 'jotai/utils';

const countAtom = atomWithDefault(() => 0);

// Reset to default
const [, setCount] = useAtom(countAtom);
setCount(undefined); // Resets to 0
```

## Utils

### atomWithReset

```typescript
import { atomWithReset, useResetAtom } from 'jotai/utils';

const countAtom = atomWithReset(0);

function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const reset = useResetAtom(countAtom);

  return (
    <>
      <span>{count}</span>
      <button onClick={() => setCount((c) => c + 1)}>+</button>
      <button onClick={reset}>Reset</button>
    </>
  );
}
```

### selectAtom

Create a derived atom that only updates when selected value changes.

```typescript
import { selectAtom } from 'jotai/utils';

const userAtom = atom({ name: 'John', age: 30 });
const nameAtom = selectAtom(userAtom, (user) => user.name);
```

## Provider-less Mode

Jotai works without a Provider. Each component tree gets its own store. Use `Provider` when you need a scoped store (e.g., tests).

```typescript
import { Provider } from 'jotai';

function App() {
  return (
    <Provider>
      <YourApp />
    </Provider>
  );
}
```

## TypeScript

Full TypeScript support. Atoms infer types from initial/default values.

```typescript
interface User {
  id: string;
  name: string;
}

const userAtom = atom<User | null>(null);
const usersAtom = atom<User[]>([]);
```

## When to Use Jotai

**Use when:**
- Fine-grained reactivity
- Derived/computed state from multiple sources
- Minimal re-renders matter
- Prefer bottom-up composition

**Avoid when:**
- Need Redux-style single store + reducers
- Team prefers Zustand simplicity
- Need middleware (logging, persistence) — use utils instead
