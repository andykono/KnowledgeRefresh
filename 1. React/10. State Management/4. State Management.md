# State Management

[[../README.md]]

---

Comprehensive guide to state management in React applications. Covers built-in solutions, popular libraries, and when to use each approach.

## Overview

State in React applications falls into three categories:

| Type | Description | Examples | Primary Tool |
| :-- | :-- | :-- | :-- |
| **UI State** | Local, transient | Modals, form inputs, hover | `useState` |
| **Client State** | App-wide, persistent in session | Theme, auth, preferences | Zustand, Redux, Jotai |
| **Server State** | From API, cached | Users, products, posts | TanStack Query |

---

## Decision Tree

```
What kind of state?
│
├─ UI state (modal, form, toggle)
│  └─ useState
│
├─ Shared across few components?
│  ├─ Lifting state up
│  └─ useContext
│
├─ Server/API data?
│  └─ TanStack Query
│
├─ Complex workflows (checkout, wizard)?
│  └─ XState
│
└─ Global client state?
   ├─ Simple, small app → Zustand
   ├─ Fine-grained, atomic → Jotai
   └─ Enterprise, middleware → Redux Toolkit
```

## When to Use What

### useState + useReducer + useContext
- **Use when:** Local UI state, forms, simple shared state across a few components
- **Avoid when:** Deep prop drilling, many unrelated components need same state

### Redux Toolkit
- **Use when:** Large team, complex state flows, need middleware (logging, analytics), time-travel debugging
- **Avoid when:** Small app, rapid prototyping, team unfamiliar with Redux

### Zustand
- **Use when:** Global state without boilerplate, hooks-first, fast iteration
- **Avoid when:** Need Redux middleware ecosystem, very complex reducer logic

### Jotai
- **Use when:** Bottom-up state, derived/computed state, minimal re-renders
- **Avoid when:** Prefer single store, Redux-style architecture

### XState
- **Use when:** Checkout flows, wizards, auth flows, any state with strict transitions
- **Avoid when:** Simple toggles, straightforward CRUD

### TanStack Query
- **Use when:** Any server/API data — caching, refetching, mutations
- **Avoid when:** Purely client-side state

## Recommended Combinations

**Small project:** `useState` + `useContext`

**Medium project:** `Zustand` (client) + `TanStack Query` (server)

**Large project:** `Redux Toolkit` or `Zustand` (client) + `TanStack Query` (server) + `XState` (complex flows)

---

## Key Principles

1. **Separate server and client state** — Use TanStack Query for API data, not Redux/Zustand. Use client state libraries for theme, auth, UI preferences.
2. **Start simple** — Use `useState` and `useContext` until you need more. Add external libraries when prop drilling or re-render issues appear.
3. **Colocate state** — Keep state close to where it's used. Lift only when necessary.
