# XState

[[4. State Management.md]]

---

XState implements state machines and statecharts. Use it when state has strict transitions, complex workflows, or you want to prevent invalid states.

## Installation

```bash
npm install xstate @xstate/react
```

## State Machines

A state machine has:
- **States:** Enumerated, explicit states
- **Transitions:** Events that move between states
- **Context:** Extended state (data)
- **Actions:** Side effects on transitions
- **Guards:** Conditional transitions

## createMachine

```typescript
import { createMachine } from 'xstate';

const toggleMachine = createMachine({
  id: 'toggle',
  initial: 'off',
  states: {
    off: {
      on: { TOGGLE: 'on' },
    },
    on: {
      on: { TOGGLE: 'off' },
    },
  },
});
```

## useMachine

```typescript
import { useMachine } from '@xstate/react';

function Toggle() {
  const [state, send] = useMachine(toggleMachine);

  return (
    <button onClick={() => send('TOGGLE')}>
      {state.value === 'on' ? 'ON' : 'OFF'}
    </button>
  );
}
```

## Context

Store data alongside the machine state. Use `assign` to update context.

```typescript
import { createMachine, assign } from 'xstate';

const counterMachine = createMachine({
  context: { count: 0 },
  initial: 'idle',
  states: {
    idle: {
      on: {
        INCREMENT: {
          actions: assign({ count: ({ context }) => context.count + 1 }),
        },
        DECREMENT: {
          actions: assign({ count: ({ context }) => context.count - 1 }),
        },
      },
    },
  },
});
```

## Guards

Conditional transitions. Only transition if guard returns true.

```typescript
const machine = createMachine({
  context: { count: 0 },
  initial: 'active',
  states: {
    active: {
      on: {
        DECREMENT: [
          {
            target: 'active',
            guard: ({ context }) => context.count > 0,
            actions: assign({ count: ({ context }) => context.count - 1 }),
          },
          {
            target: 'empty',
            guard: ({ context }) => context.count <= 0,
          },
        ],
      },
    },
    empty: {
      on: {
        INCREMENT: {
          target: 'active',
          actions: assign({ count: 1 }),
        },
      },
    },
  },
});
```

## Actions

Run side effects on transitions. Use `entry` and `exit` for state entry/exit.

```typescript
const machine = createMachine({
  initial: 'loading',
  states: {
    loading: {
      entry: 'fetchData',
      on: {
        SUCCESS: 'loaded',
        ERROR: 'error',
      },
    },
    loaded: {
      entry: 'logLoaded',
    },
    error: {},
  },
}, {
  actions: {
    fetchData: ({ context }) => {
      // Trigger async fetch
    },
    logLoaded: () => console.log('Data loaded'),
  },
});
```

## Services (Invoked)

Handle async logic. Machine waits for service to resolve.

```typescript
import { createMachine, assign } from 'xstate';

const fetchMachine = createMachine({
  id: 'fetch',
  initial: 'loading',
  context: { data: null, error: null },
  states: {
    loading: {
      invoke: {
        src: 'fetchData',
        onDone: {
          target: 'success',
          actions: assign({ data: (_, event) => event.data }),
        },
        onError: {
          target: 'failure',
          actions: assign({ error: (_, event) => event.data }),
        },
      },
    },
    success: {},
    failure: {
      on: { RETRY: 'loading' },
    },
  },
}, {
  services: {
    fetchData: async () => {
      const res = await fetch('/api/data');
      return res.json();
    },
  },
});
```

## Parallel States

Multiple states active at once. Useful for independent concerns.

```typescript
const machine = createMachine({
  type: 'parallel',
  states: {
    mode: {
      initial: 'view',
      states: {
        view: { on: { EDIT: 'edit' } },
        edit: { on: { SAVE: 'view', CANCEL: 'view' } },
      },
    },
    validation: {
      initial: 'valid',
      states: {
        valid: { on: { INVALID: 'invalid' } },
        invalid: { on: { VALID: 'valid' } },
      },
    },
  },
});
```

## TypeScript Typegen

Generate types from the machine for type-safe `send` and `context`.

```typescript
import { createMachine } from 'xstate';

const machine = createMachine({
  types: {} as {
    context: { count: number };
    events: { type: 'INCREMENT' } | { type: 'DECREMENT' };
  },
  context: { count: 0 },
  initial: 'idle',
  states: {
    idle: {
      on: {
        INCREMENT: { actions: 'inc' },
        DECREMENT: { actions: 'dec' },
      },
    },
  },
}, {
  actions: {
    inc: ({ context }) => ({ ...context, count: context.count + 1 }),
    dec: ({ context }) => ({ ...context, count: context.count - 1 }),
  },
});
```

## Practical Example: Auth Flow

```typescript
const authMachine = createMachine({
  id: 'auth',
  initial: 'checking',
  context: { user: null as User | null },
  states: {
    checking: {
      invoke: {
        src: 'checkAuth',
        onDone: { target: 'authenticated', actions: assign({ user: (_, e) => e.data }) },
        onError: 'unauthenticated',
      },
    },
    unauthenticated: {
      on: { LOGIN: 'authenticating' },
    },
    authenticating: {
      invoke: {
        src: 'login',
        onDone: { target: 'authenticated', actions: assign({ user: (_, e) => e.data }) },
        onError: 'unauthenticated',
      },
    },
    authenticated: {
      on: { LOGOUT: 'unauthenticated' },
    },
  },
}, {
  services: {
    checkAuth: async () => (await fetch('/api/me')).json(),
    login: async (_, event) => (await fetch('/api/login', { body: JSON.stringify(event) })).json(),
  },
});
```

## When to Use XState

**Use when:**
- Checkout wizards, multi-step forms
- Auth flows (login, logout, session)
- Complex UI (modals, drawers with nested states)
- Any flow where invalid states must be impossible

**Avoid when:**
- Simple toggles (useState is enough)
- CRUD with no complex transitions
- Team unfamiliar with state machines
