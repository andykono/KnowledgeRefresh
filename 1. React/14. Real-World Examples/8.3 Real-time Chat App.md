# Real-time Chat App Example

[[8. Real-World Examples.md]]

Real-time messaging application with WebSocket, complex state, and user interactions.

## Features

- User authentication and profiles
- Chat rooms and direct messaging
- Real-time message updates (WebSocket)
- User presence indicators
- Message history
- Typing indicators
- File sharing
- Notifications

## Architecture

```
Real-time Communication:
├── WebSocket (message events)
├── Event handling (subscriptions)
└── Connection state machine

State Management:
├── Zustand (chat state)
├── Context (auth)
└── XState (connection states)

Features:
├── Authentication
├── ChatRooms
├── DirectMessages
├── UserPresence
├── Notifications
└── FileUpload
```

## Advanced Patterns

- **Observer Pattern:** Message subscriptions
- **Command Pattern:** Message queue for offline support
- **State Machine (XState):** Connection states (connecting, connected, error)
- **Decorator Pattern:** Message encryption/decryption
- **Facade:** WebSocket API simplification
- **Custom Hooks:** 
  - useWebSocket (connection management)
  - useMessages (message history)
  - usePresence (user status)

## WebSocket Connection

```typescript
// Connection state machine
enum ConnectionState {
  DISCONNECTED = 'disconnected',
  CONNECTING = 'connecting',
  CONNECTED = 'connected',
  ERROR = 'error',
  RECONNECTING = 'reconnecting',
}

// XState machine for explicit state management
const connectionMachine = createMachine({
  initial: 'disconnected',
  states: {
    disconnected: {
      on: { CONNECT: 'connecting' }
    },
    connecting: {
      on: { 
        SUCCESS: 'connected',
        ERROR: 'error'
      }
    },
    connected: {
      on: { 
        DISCONNECT: 'disconnected',
        ERROR: 'error'
      }
    },
    error: {
      on: { RETRY: 'connecting' }
    }
  }
});
```
