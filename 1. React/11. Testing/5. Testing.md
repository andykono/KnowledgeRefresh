# 5. Testing

[[../README.md]]

A comprehensive guide to testing React and TypeScript applications—from unit tests to end-to-end flows.

---

## Testing Pyramid

```
           /\
          /  \     E2E Tests (Playwright, Cypress)
         /    \    5–10% — Critical user journeys
        /------\
       /        \   Integration Tests
      /          \  20–30% — Components + Context/Router/API
     /            \
    /              \ Unit Tests
   /________________\ 60–80% — Pure functions, reducers, hooks
```

**Principle:** Many fast unit tests, fewer integration tests, few slow E2E tests.

---

## Overview Table

| Level | Tool | What to Test | Speed |
| :-- | :-- | :-- | :-- |
| **Unit** | Jest / Vitest | Pure functions, reducers, custom hooks | Fast |
| **Component** | React Testing Library | UI rendering, user interactions | Fast |
| **Integration** | RTL + MSW | Context, Router, API flows | Medium |
| **E2E** | Playwright | Full browser, real user flows | Slow |

---

## Decision Guide: Which Test Type?

| Scenario | Test Type |
| :-- | :-- |
| Pure function (e.g. `formatDate`, `validateEmail`) | Unit |
| Reducer or state machine | Unit |
| Custom hook in isolation | Unit |
| Component renders and responds to clicks | Component |
| Form validation and submission | Component / Integration |
| Component needs AuthContext | Integration |
| Component needs React Router | Integration |
| Login → Dashboard → CRUD flow | Integration / E2E |
| Full checkout flow in browser | E2E |
| Cross-browser compatibility | E2E |

## Quick Setup

### Vitest + React Testing Library

```bash
npm install -D vitest @testing-library/react @testing-library/jest-dom @testing-library/user-event jsdom
```

```typescript
// vitest.config.ts
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    setupFiles: ['./src/test/setup.ts'],
    globals: true,
  },
});
```

```typescript
// src/test/setup.ts
import '@testing-library/jest-dom';
```

### Jest Alternative

```bash
npm install -D jest @testing-library/react @testing-library/jest-dom jest-environment-jsdom ts-jest
```

```typescript
// jest.config.js
module.exports = {
  testEnvironment: 'jsdom',
  setupFilesAfterEnv: ['<rootDir>/src/test/setup.ts'],
  moduleNameMapper: { '\\.(css|less)$': 'identity-obj-proxy' },
};
```
