# React Patterns

[[../README.md]]

React-специфичные паттерны и подходы к организации кода. Это паттерны, которые родились в экосистеме React и работают с её особенностями.

---

## What Are React Patterns?

React паттерны:
- ✅ Используют особенности React (hooks, context, JSX)
- ✅ Решают проблемы, специфичные для React
- ✅ Часто комбинируют Design Patterns с React API
- ✅ Эволюционируют вместе с React

---

## Why React Patterns Matter

```typescript
// ❌ Before: Complicated component with mixed concerns
class UserProfile extends React.Component {
  state = { user: null, loading: false };
  componentDidMount() { this.fetchUser(); }
  componentDidUpdate() { this.fetchUser(); }
  fetchUser = async () => { /* ... */ };
  render() { /* 200 lines */ }
}

// ✅ After: Clean, composable hooks-based approach
function UserProfile() {
  const { user, loading } = useUser();
  return <div>{user?.name}</div>;
}
```

## Core React Patterns

### [[9.1 Compound Components]]
**Pattern:** Components that work together, with shared state

Best for: Flexible, customizable components (Tabs, Accordion, Forms)

```typescript
<Tabs>
  <Tabs.List>
    <Tabs.Item>Tab 1</Tabs.Item>
  </Tabs.List>
  <Tabs.Content>Content 1</Tabs.Content>
</Tabs>
```

---

### [[9.2 Render Props, HOCs, Hooks]]
**Pattern:** Share logic between components (three approaches)

Best for: Reusing stateful logic

```typescript
// Render Props
<WithUser>{(user) => <div>{user.name}</div>}</WithUser>

// HOC
const Connected = withUser(MyComponent);

// Hooks (Modern)
const user = useUser();
```

---

### [[9.3 Custom Hooks Patterns]]
**Pattern:** Reuse logic through hooks composition

Best for: Encapsulating and sharing logic

```typescript
function useAsync<T>(fn: () => Promise<T>) { /* ... */ }
function useDebounce<T>(value: T, delay: number) { /* ... */ }
function usePrevious<T>(value: T) { /* ... */ }
```

---

### [[9.4 Context API Patterns]]
**Pattern:** Share state across component tree without props

Best for: Theme, auth, user preferences, language

```typescript
const ThemeContext = createContext();
function ThemeProvider({ children }) { /* ... */ }
function useTheme() { return useContext(ThemeContext); }
```

## Pattern Selection Matrix

| Need | Pattern | When | Why |
| :-- | :-- | :-- | :-- |
| Component composition | Compound Components | Customizable components | Flexible + maintainable |
| Share state | Custom Hook | Multiple components | Reusable + clean |
| Global state | Context API | App-wide values | Simple + built-in |
| Complex logic | Custom Hook | Logic reuse | Separation of concerns |
| Complex UI state | State Machine (XState) | Multi-state UI | Explicit + predictable |

## Common Use Cases

### Theme Switching
```typescript
// Pattern: Context API + Custom Hook
const useTheme = () => useContext(ThemeContext);
const user = useTheme(); // Use anywhere
```

### Form Validation
```typescript
// Pattern: Custom Hook
const form = useForm({ email, password });
form.validate(); // Returns errors
```

### Data Fetching
```typescript
// Pattern: Custom Hook
const { data, loading, error } = useAsync(() => api.getUsers());
```

### Multiple Related State
```typescript
// Pattern: useReducer
const [state, dispatch] = useReducer(reducer, initialState);
```

## Advanced Topics

### Hooks Rules
```typescript
// ✅ DO
useEffect(() => { /* ... */ }, [deps]);  // Inside component or custom hook
const hook = useCustomHook();             // At top level

// ❌ DON'T
if (condition) useEffect(...);            // Conditional hooks
hooks.push(useEffect);                    // Array of hooks
```

### Performance Optimization
```typescript
useMemo(() => expensiveComputation(), [deps]);    // Memoize value
useCallback(() => handleClick(), [deps]);         // Memoize function
React.memo(Component);                             // Memoize component
```

### Error Boundaries
```typescript
class ErrorBoundary extends React.Component {
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  render() {
    if (this.state.hasError) return <ErrorFallback />;
    return this.props.children;
  }
}
```

## React Patterns Evolution

```
Class Components (2015)
  ↓
Render Props & HOCs (2017)
  ↓
Hooks (2019) ← Modern standard
  ↓
Server Components (2023) ← Future
```

## Quick Comparison

| Approach | Pro | Con | Use When |
| :-- | :-- | :-- | :-- |
| **Class Components** | Full featured | Verbose | Legacy code |
| **Functional + Hooks** | Clean, composable | Learning curve | New projects |
| **Compound Components** | Flexible | Complex | Customizable UI |
| **Context API** | Built-in, simple | Prop drilling still possible | Small-medium apps |
| **State Machines** | Explicit, predictable | Extra package | Complex flows |
