# Render Props vs HOCs vs Hooks

[[9. React Patterns.md]]

Три подхода к **reusing stateful logic** между компонентами. Каждый имеет свои плюсы и минусы.

---

## Approach 1: Render Props

**Idea:** Pass a function as a prop that returns JSX.

```typescript
// Component with logic
interface WithUserProps {
  children: (user: User | null, loading: boolean, error: Error | null) => React.ReactNode;
}

function WithUser({ children }: WithUserProps) {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setLoading(true);
    api.getUser()
      .then(setUser)
      .catch(setError)
      .finally(() => setLoading(false));
  }, []);

  return children(user, loading, error);
}

// Usage 1: Display user name
function Component1() {
  return (
    <WithUser>
      {(user, loading, error) => (
        <>
          {loading && <div>Loading...</div>}
          {error && <div>Error: {error.message}</div>}
          {user && <div>{user.name}</div>}
        </>
      )}
    </WithUser>
  );
}

// Usage 2: Display user email
function Component2() {
  return (
    <WithUser>
      {(user, loading, error) => (
        <>
          {loading && <span>Fetching...</span>}
          {error && <span>Failed</span>}
          {user && <span>{user.email}</span>}
        </>
      )}
    </WithUser>
  );
}
```

### Pros & Cons

✅ **Pros:**
- Clear data flow (you see what comes from the prop)
- Can render multiple children with different logic
- Easy to debug

❌ **Cons:**
- Can lead to "callback hell"
- Performance issues (function recreated each render)
- Verbose syntax

## Approach 2: Higher-Order Components (HOC)

**Idea:** Wrap a component to inject props/state.

```typescript
// HOC that wraps a component
interface WithUserProps {
  user: User | null;
  loading: boolean;
  error: Error | null;
}

function withUser<P extends WithUserProps>(
  Component: React.ComponentType<P>
): React.FC<Omit<P, keyof WithUserProps>> {
  return function WithUserComponent(props: any) {
    const [user, setUser] = useState<User | null>(null);
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState<Error | null>(null);

    useEffect(() => {
      setLoading(true);
      api.getUser()
        .then(setUser)
        .catch(setError)
        .finally(() => setLoading(false));
    }, []);

    return (
      <Component
        {...props}
        user={user}
        loading={loading}
        error={error}
      />
    );
  };
}

// Usage 1
function MyComponent1({ user, loading, error }: WithUserProps) {
  return (
    <>
      {loading && <div>Loading...</div>}
      {error && <div>Error</div>}
      {user && <div>{user.name}</div>}
    </>
  );
}
export const Enhanced1 = withUser(MyComponent1);

// Usage 2
function MyComponent2({ user, loading, error }: WithUserProps) {
  return <h1>{user?.email}</h1>;
}
export const Enhanced2 = withUser(MyComponent2);

// In JSX
<Enhanced1 />
<Enhanced2 />
```

### Pros & Cons

✅ **Pros:**
- Clean component signature
- Reusable across many components
- Works with class components too

❌ **Cons:**
- Wrapper hell (many HOCs nested)
- Props naming conflicts
- Static methods not copied
- Harder to debug (React DevTools shows HOCs)

## Approach 3: Custom Hooks (Modern ⭐)

**Idea:** Extract logic into a reusable hook.

```typescript
// Custom hook with logic
function useUser() {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setLoading(true);
    api.getUser()
      .then(setUser)
      .catch(setError)
      .finally(() => setLoading(false));
  }, []);

  return { user, loading, error };
}

// Usage 1
function Component1() {
  const { user, loading, error } = useUser();
  return (
    <>
      {loading && <div>Loading...</div>}
      {error && <div>Error</div>}
      {user && <div>{user.name}</div>}
    </>
  );
}

// Usage 2
function Component2() {
  const { user } = useUser();
  return <h1>{user?.email}</h1>;
}

// Usage 3: Even in another hook!
function useUserName() {
  const { user } = useUser();
  return user?.name;
}
```

### Pros & Cons

✅ **Pros:**
- Simplest syntax
- Easy to compose hooks
- Great DevTools support
- No wrapper hell

❌ **Cons:**
- Can only use in functional components
- Can't use conditionally (Rules of Hooks)
- Requires React 16.8+

## Comparison

| Aspect | Render Props | HOC | Custom Hook |
| :-- | :-- | :-- | :-- |
| **Syntax** | 😞 Complex | 😐 Medium | ⭐ Simple |
| **Performance** | ⚠️ Can be slow | 😐 Medium | ⭐ Best |
| **Composability** | 😞 Callback hell | 😐 Wrapper hell | ⭐ Easy |
| **DevTools** | 😞 Poor | 😞 Hard to debug | ⭐ Clear |
| **Learning Curve** | 😞 High | 😐 Medium | ⭐ Low |
| **Class Components** | ✅ Yes | ✅ Yes | ❌ No |
| **Conditional** | ✅ Yes | ✅ Yes | ❌ No (Rules) |
| **Recommended** | ❌ Legacy | ⚠️ Sometimes | ⭐ Modern |

## Real-World Comparison

### Problem: Share user authentication state

#### With Render Props
```typescript
<WithAuth>
  {(user, isAuth, login, logout) => (
    <>
      {isAuth ? (
        <button onClick={logout}>Logout {user.name}</button>
      ) : (
        <button onClick={login}>Login</button>
      )}
    </>
  )}
</WithAuth>
```

#### With HOC
```typescript
function Navbar({ user, isAuth, login, logout }: WithAuthProps) {
  return (
    <>
      {isAuth ? (
        <button onClick={logout}>Logout {user.name}</button>
      ) : (
        <button onClick={login}>Login</button>
      )}
    </>
  );
}

export const AuthedNavbar = withAuth(Navbar);

// Usage
<AuthedNavbar />
```

#### With Custom Hook ⭐
```typescript
function Navbar() {
  const { user, isAuth, login, logout } = useAuth();
  return (
    <>
      {isAuth ? (
        <button onClick={logout}>Logout {user.name}</button>
      ) : (
        <button onClick={login}>Login</button>
      )}
    </>
  );
}
```

## Advanced: Composing Multiple Hooks

```typescript
// Hook 1: Fetch data
function useFetch<T>(url: string) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setLoading(true);
    fetch(url)
      .then(r => r.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}

// Hook 2: Debounce
function useDebounce<T>(value: T, delay: number) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
}

// Hook 3: Combine them
function useSearchUsers(query: string) {
  const debouncedQuery = useDebounce(query, 500);
  const { data: results, loading } = useFetch<User[]>(
    `/api/users?q=${debouncedQuery}`
  );
  return { results, loading };
}

// Usage
function SearchUsers() {
  const [query, setQuery] = useState('');
  const { results, loading } = useSearchUsers(query);

  return (
    <>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search..."
      />
      {loading && <div>Searching...</div>}
      <ul>
        {results?.map(user => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
}
```

## When to Use Each?

### Use Custom Hooks When:
- ✅ Building new code (modern React)
- ✅ Sharing logic between multiple components
- ✅ Need simple, clean syntax
- ✅ Don't need to wrap class components

### Use HOCs When:
- ⚠️ Need to wrap existing class components
- ⚠️ Working in legacy codebases
- ⚠️ Need advanced prop manipulation

### Use Render Props When:
- ⚠️ Very rare edge cases
- ⚠️ Working with libraries that use it (like React Router old version)
- ⚠️ Need explicit control over what you render

## Migration Path

If you have old code:

```typescript
// Old: HOC approach
const Enhanced = withUser(MyComponent);

// New: Convert to hook
function useUser() { /* ... */ }
function MyComponent() {
  const user = useUser();
  // ...
}
```

---

## Best Practices

✅ **Do:**
- Use custom hooks for new code
- Name hooks with `use` prefix
- Export both hook and component separately
- Type your hooks properly

❌ **Don't:**
- Don't use `withXX` HOCs unless necessary
- Don't nest HOCs deeply
- Don't use render props in new projects
- Don't pass inline functions to props (performance)
