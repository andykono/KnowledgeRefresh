# Streaming and Suspense

[[6. Server Components.md]]

---

Streaming Server-Side Rendering (SSR) sends HTML in chunks as it becomes ready. React's `Suspense` defines boundaries where the UI can show fallbacks while async work completes. Together they enable progressive loading and selective hydration.

## Streaming SSR

Instead of waiting for the entire page to render, the server streams HTML as each part is ready:

```
[Initial shell] → [Suspense boundary 1] → [Suspense boundary 2] → ...
```

The browser can paint and hydrate incrementally.

## Suspense Boundaries

`Suspense` wraps async components (or components that use `use()`) and shows a fallback until they resolve:

```typescript
import { Suspense } from 'react';

async function SlowComponent() {
  const data = await fetchData();  // Simulated slow fetch
  return <div>{data}</div>;
}

export default function Page() {
  return (
    <div>
      <h1>Page Title</h1>
      <Suspense fallback={<div>Loading...</div>}>
        <SlowComponent />
      </Suspense>
    </div>
  );
}
```

### Multiple Boundaries

Each boundary streams independently. Fast content appears first:

```typescript
export default function Page() {
  return (
    <div>
      <Suspense fallback={<HeaderSkeleton />}>
        <Header />
      </Suspense>

      <main>
        <Suspense fallback={<ArticleSkeleton />}>
          <Article />
        </Suspense>

        <aside>
          <Suspense fallback={<SidebarSkeleton />}>
            <Sidebar />
          </Suspense>
        </aside>
      </main>
    </div>
  );
}
```

## Loading States

### Route-Level loading.tsx (Next.js)

Create `loading.tsx` in a route segment. Next.js wraps the segment in Suspense automatically:

```
app/
  dashboard/
    loading.tsx   ← Shown while dashboard layout/page loads
    page.tsx
    layout.tsx
```

```typescript
// app/dashboard/loading.tsx
export default function DashboardLoading() {
  return (
    <div className="animate-pulse">
      <div className="h-8 bg-gray-200 rounded w-1/3 mb-4" />
      <div className="h-4 bg-gray-200 rounded w-full mb-2" />
      <div className="h-4 bg-gray-200 rounded w-5/6" />
    </div>
  );
}
```

### Inline Fallbacks

```typescript
<Suspense fallback={
  <div className="flex gap-2">
    {[1, 2, 3].map(i => (
      <div key={i} className="w-32 h-48 bg-gray-200 rounded animate-pulse" />
    ))}
  </div>
}>
  <ProductGrid />
</Suspense>
```

## Selective Hydration

Client Components inside Suspense boundaries hydrate in order of streamed content. The browser can:
1. Show static content immediately
2. Hydrate interactive parts as they arrive
3. Defer low-priority components

```typescript
export default function Page() {
  return (
    <>
      <StaticContent />
      <Suspense fallback={<Spinner />}>
        <InteractiveWidget />  {/* Hydrates when ready */}
      </Suspense>
      <Suspense fallback={null}>
        <BelowFoldContent />   {/* Lower priority */}
      </Suspense>
    </>
  );
}
```

## use() in Client Components

React 19's `use()` lets Client Components unwrap promises. Combined with Suspense, the parent provides the boundary:

```typescript
"use client";

import { use } from 'react';

function ClientProduct({ productPromise }: { productPromise: Promise<Product> }) {
  const product = use(productPromise);  // Suspends until resolved
  return <div>{product.name}</div>;
}

// Server Component
export default function Page() {
  const productPromise = fetchProduct(id);
  return (
    <Suspense fallback={<ProductSkeleton />}>
      <ClientProduct productPromise={productPromise} />
    </Suspense>
  );
}
```

## Nested Suspense

Inner boundaries can have their own fallbacks. The outer fallback shows until the first child resolves; inner fallbacks show for nested async work:

```typescript
<Suspense fallback={<PageSkeleton />}>
  <Header />
  <Suspense fallback={<ListSkeleton />}>
    <ProductList />
  </Suspense>
  <Suspense fallback={<ChartSkeleton />}>
    <AnalyticsChart />
  </Suspense>
</Suspense>
```

## Error Boundaries with Suspense

Use Error Boundaries to catch failures in async components:

```typescript
'use client';

import { Component, ReactNode } from 'react';

export class ErrorBoundary extends Component<
  { children: ReactNode; fallback: ReactNode },
  { hasError: boolean }
> {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback;
    }
    return this.props.children;
  }
}

// Usage
<ErrorBoundary fallback={<ErrorMessage />}>
  <Suspense fallback={<Loading />}>
    <AsyncComponent />
  </Suspense>
</ErrorBoundary>
```

## Best Practices

### Granular Boundaries

Place Suspense around the smallest async unit that makes sense. Avoid one giant boundary for the whole page.

```typescript
// ✅ Good: each section streams independently
<Suspense fallback={<NavSkeleton />}><Nav /></Suspense>
<Suspense fallback={<ContentSkeleton />}><Content /></Suspense>
<Suspense fallback={<FooterSkeleton />}><Footer /></Suspense>

// ❌ Avoid: entire page waits for slowest part
<Suspense fallback={<FullPageSkeleton />}>
  <Nav />
  <Content />
  <Footer />
</Suspense>
```

### Meaningful Fallbacks

Match the fallback to the final layout to reduce layout shift:

```typescript
<Suspense fallback={
  <div className="grid grid-cols-3 gap-4">
    {[1, 2, 3].map(i => <CardSkeleton key={i} />)}
  </div>
}>
  <ProductGrid />
</Suspense>
```

### Avoid Waterfalls

Fetch in parallel when possible. Sequential awaits create waterfalls:

```typescript
// ❌ Waterfall: B waits for A
async function Page() {
  const a = await fetchA();
  const b = await fetchB(a.id);  // Waits for A
  return <Content a={a} b={b} />;
}

// ✅ Parallel: A and B fetch together
async function Page() {
  const [a, b] = await Promise.all([fetchA(), fetchB()]);
  return <Content a={a} b={b} />;
}
```
