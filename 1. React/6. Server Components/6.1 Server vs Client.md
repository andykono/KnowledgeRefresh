# Server vs Client Components

[[6. Server Components.md]]

---

React Server Components (RSC) run on the server by default. Client Components (`"use client"`) run on both server and client. Understanding when and how to use each is essential for Next.js App Router and similar frameworks.

## The Directives

### `"use client"`

Marks a component (and its imports) as a **Client Component**. It will:
- Be rendered on the server for initial HTML (SSR)
- Be hydrated on the client
- Support hooks, event handlers, browser APIs
- Be included in the JavaScript bundle

```typescript
"use client";

import { useState } from 'react';

export function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(c => c + 1)}>
      Count: {count}
    </button>
  );
}
```

### No Directive (Default)

Components without `"use client"` are **Server Components**:
- Run only on the server
- Cannot use `useState`, `useEffect`, `onClick`, etc.
- Can be `async` and `await` data directly
- Not in the client bundle

```typescript
// Server Component (no directive)
async function ProductList() {
  const products = await db.products.findMany();
  return (
    <ul>
      {products.map(p => (
        <li key={p.id}>{p.name}</li>
      ))}
    </ul>
  );
}
```

### `"use server"`

Marks a function as a **Server Action**. It runs on the server when called from the client.

```typescript
"use server";

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string;
  await db.posts.create({ data: { title } });
  revalidatePath('/posts');
}
```

## When to Use Each

### Use Server Components When You Need To

- Fetch data (no need for `useEffect` + `useState`)
- Access backend resources (DB, file system, secrets)
- Keep large dependencies on server (e.g. heavy libs)
- Reduce client bundle size

### Use Client Components When You Need To

- Use React hooks (`useState`, `useEffect`, `useContext`, etc.)
- Add event handlers (`onClick`, `onChange`, etc.)
- Use browser APIs (`localStorage`, `window`, etc.)
- Use third-party client-only libraries

## Composition Patterns

### Server Component → Client Component

Server Components can render Client Components and pass serializable props:

```typescript
// Server Component
async function Page() {
  const data = await fetchData();
  return <ClientChart data={data} />;  // ✅ data is serializable
}

// Client Component
"use client";
function ClientChart({ data }: { data: ChartData[] }) {
  const [hovered, setHovered] = useState<number | null>(null);
  return <Chart data={data} onHover={setHovered} />;
}
```

### Client Component → Server Component (No)

Client Components cannot import and render Server Components directly:

```typescript
// ❌ WRONG
"use client";
import { ServerList } from './ServerList';  // Server Component
function ClientWrapper() {
  return <ServerList />;  // Error: can't render server from client
}
```

### Passing Server Components as Children

You can pass Server Components as `children` to Client Components:

```typescript
// Server Component
async function Layout({ children }: { children: React.ReactNode }) {
  const nav = await getNav();
  return (
    <div>
      <ClientSidebar nav={nav}>
        {children}  {/* children can be Server Components */}
      </ClientSidebar>
    </div>
  );
}

// Client Component
"use client";
function ClientSidebar({ nav, children }: { nav: Nav[]; children: React.ReactNode }) {
  const [open, setOpen] = useState(false);
  return (
    <aside>
      <button onClick={() => setOpen(!open)}>Toggle</button>
      {open && <nav>{/* ... */}</nav>}
      {children}
    </aside>
  );
}
```

### Slot Pattern

Pass Server Components as named props (slots):

```typescript
// Server Component
async function Page() {
  const sidebar = await getSidebar();
  const content = await getContent();
  return (
    <ClientLayout
      sidebar={<ServerSidebar data={sidebar} />}
      content={<ServerContent data={content} />}
    />
  );
}

// Client Component
"use client";
function ClientLayout({ sidebar, content }: { sidebar: React.ReactNode; content: React.ReactNode }) {
  return (
    <div>
      <aside>{sidebar}</aside>
      <main>{content}</main>
    </div>
  );
}
```

## Data Flow

```
Server Component (async)
  ↓ await fetch / db
  ↓ pass data as props
Client Component
  ↓ use hooks, events
  ↓ call Server Action
Server Action
  ↓ mutate, revalidate
```

### Fetching in Server Components

```typescript
async function UserProfile({ userId }: { userId: string }) {
  const user = await db.user.findUnique({ where: { id: userId } });
  if (!user) return null;
  return (
    <div>
      <h1>{user.name}</h1>
      <ClientEditButton user={user} />
    </div>
  );
}
```

### Passing Functions

You cannot pass regular functions from Server to Client. Use Server Actions:

```typescript
// ❌ WRONG
<ClientButton onClick={() => doSomething()} />  // Function not serializable

// ✅ CORRECT
"use server";
async function doSomething() { /* ... */ }

<ClientButton action={doSomething} />  // Server Action is serializable
```

---

## Serialization Rules

Props passed from Server to Client must be serializable:

| ✅ Allowed | ❌ Not Allowed |
| :-- | :-- |
| Primitives (string, number, boolean) | Functions (except Server Actions) |
| Plain objects, arrays | Class instances |
| Date, Map, Set (serialized) | Symbols |
| JSX (as children or props) | `undefined` (use `null`) |
| Server Actions | Circular references |

---

## Boundary Placement

Put `"use client"` as low as possible to keep more of the tree as Server Components:

```typescript
// ✅ Good: only interactive part is client
function Page() {
  return (
    <div>
      <ServerHeader />
      <ServerContent />
      <ClientAddToCart />  {/* Only this needs client */}
    </div>
  );
}

// ❌ Avoid: entire page is client
"use client";
function Page() {
  return (
    <div>
      <Header />
      <Content />
      <AddToCart />
    </div>
  );
}
```
